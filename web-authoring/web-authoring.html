Student Pane – Web Authoring (IGCSE ICT)

1. What is Web Authoring?

Web authoring is the process of designing and creating web pages that can be viewed using a web browser. In Cambridge IGCSE ICT, web authoring focuses on creating structured, well-designed, and accessible web pages using HTML for structure and CSS for presentation.

Students are assessed on their ability to:

  • Create valid HTML pages
  • Apply external CSS stylesheets
  • Use images, links, and navigation correctly
  • Design pages suitable for a target audience

2. Basic Structure of an HTML Webpage

Every HTML webpage must follow a clear structure. This structure helps browsers understand how to display the page and helps developers maintain the content easily. 

<!DOCTYPE html>
<html>
<head>
  <title>My Webpage</title>
</head>
<body>

</body>
</html>

The DOCTYPE declaration tells the browser that this page is written in HTML5. Missing or incorrect structure can lead to lost marks in practical examinations.

3. Common HTML Elements You Must Know

The following elements are frequently tested in both theory and practical examinations:

  • <h1> to <h6> – headings
  • <p> – paragraphs
  • <br> – line breaks
  • <ul> and <ol> – lists
  • <a> – hyperlinks
  • <img> – images

4. Using Images Correctly

Images are added using the <img> tag. Every image must include alternative text using the alt attribute. This is essential for accessibility and is required for full marks. 

<img src="images/school.jpg" alt="Front view of the school building">

5. Introduction to CSS

CSS (Cascading Style Sheets) controls the appearance of a webpage. Cambridge IGCSE prefers the use of external CSS because it separates content from presentation and demonstrates good design practice.

An external CSS file is linked using the following line inside the <head> section: 

<link rel="stylesheet" href="style.css">

6. Accessibility and Usability

A good webpage must be accessible to all users. This includes:

  • Clear navigation
  • Readable font sizes
  • Good colour contrast
  • Alternative text for images

Accessibility and usability are common theory questions in examinations.

7. Hyperlinks and Navigation

Hyperlinks allow users to move between pages and sections of a website. In the Cambridge IGCSE ICT practical exam, students are often required to create navigation menus that link multiple pages together.

A hyperlink is created using the <a> tag: 

<a href="about.html">About Us</a>

Links can also point to sections on the same page using IDs: 

<a href="#practice">Go to Practice Pane</a>

Incorrect file paths are one of the most common reasons students lose marks in practical examinations.

8. Relative File Paths (Very Important)

A relative file path describes the location of a file in relation to the current file. Cambridge practical exams expect students to use relative paths correctly.

  • style.css – file in the same folder
  • images/logo.png – image inside an images folder
  • ../index.html – file one folder above

Using incorrect paths will result in broken images or links, which leads to lost marks.

9. Using Lists Effectively

Lists are commonly used for navigation menus, instructions, and features. There are two main types of lists:

  • Unordered lists (<ul>)
  • Ordered lists (<ol>)
<ul>
  <li>Home</li>
  <li>About</li>
  <li>Contact</li>
</ul>

10. Good Page Layout and Consistency

A professional website uses a consistent layout across all pages. This means the header, navigation bar, and footer should appear in the same position on every page.

Consistency improves usability and is often mentioned in theory questions.

11. Testing a Webpage

Before submitting a webpage in an exam, students should always test their work carefully.

  • Check that all links work correctly
  • Ensure images display properly
  • Confirm CSS is linked correctly
  • Check spelling and grammar

Testing is an essential step and should be mentioned when answering theory questions about website development.

12. Common Exam Mistakes to Avoid

  • Missing alternative text for images
  • Incorrect file or folder names
  • Using inline CSS instead of external CSS
  • Poor colour contrast
  • Ignoring client requirements

Avoiding these mistakes can significantly improve exam performance.
