Student Pane – Web Authoring (IGCSE ICT)

1. What is Web Authoring?

Web authoring is the process of designing and creating web pages that can be viewed using a web browser. In Cambridge IGCSE ICT, web authoring focuses on creating structured, well-designed, and accessible web pages using HTML for structure and CSS for presentation.

Students are assessed on their ability to:

  • Create valid HTML pages
  • Apply external CSS stylesheets
  • Use images, links, and navigation correctly
  • Design pages suitable for a target audience

2. Basic Structure of an HTML Webpage

Every HTML webpage must follow a clear structure. This structure helps browsers understand how to display the page and helps developers maintain the content easily. 

<!DOCTYPE html>
<html>
<head>
  <title>My Webpage</title>
</head>
<body>

</body>
</html>

The DOCTYPE declaration tells the browser that this page is written in HTML5. Missing or incorrect structure can lead to lost marks in practical examinations.

3. Common HTML Elements You Must Know

The following elements are frequently tested in both theory and practical examinations:

  • <h1> to <h6> – headings
  • <p> – paragraphs
  • <br> – line breaks
  • <ul> and <ol> – lists
  • <a> – hyperlinks
  • <img> – images

4. Using Images Correctly

Images are added using the <img> tag. Every image must include alternative text using the alt attribute. This is essential for accessibility and is required for full marks. 

<img src="images/school.jpg" alt="Front view of the school building">

5. Introduction to CSS

CSS (Cascading Style Sheets) controls the appearance of a webpage. Cambridge IGCSE prefers the use of external CSS because it separates content from presentation and demonstrates good design practice.

An external CSS file is linked using the following line inside the <head> section: 

<link rel="stylesheet" href="style.css">

6. Accessibility and Usability

A good webpage must be accessible to all users. This includes:

  • Clear navigation
  • Readable font sizes
  • Good colour contrast
  • Alternative text for images

Accessibility and usability are common theory questions in examinations.

7. Hyperlinks and Navigation

Hyperlinks allow users to move between pages and sections of a website. In the Cambridge IGCSE ICT practical exam, students are often required to create navigation menus that link multiple pages together.

A hyperlink is created using the <a> tag: 

<a href="about.html">About Us</a>

Links can also point to sections on the same page using IDs: 

<a href="#practice">Go to Practice Pane</a>

Incorrect file paths are one of the most common reasons students lose marks in practical examinations.

8. Relative File Paths (Very Important)

A relative file path describes the location of a file in relation to the current file. Cambridge practical exams expect students to use relative paths correctly.

  • style.css – file in the same folder
  • images/logo.png – image inside an images folder
  • ../index.html – file one folder above

Using incorrect paths will result in broken images or links, which leads to lost marks.

9. Using Lists Effectively

Lists are commonly used for navigation menus, instructions, and features. There are two main types of lists:

  • Unordered lists (<ul>)
  • Ordered lists (<ol>)
<ul>
  <li>Home</li>
  <li>About</li>
  <li>Contact</li>
</ul>

10. Good Page Layout and Consistency

A professional website uses a consistent layout across all pages. This means the header, navigation bar, and footer should appear in the same position on every page.

Consistency improves usability and is often mentioned in theory questions.

11. Testing a Webpage

Before submitting a webpage in an exam, students should always test their work carefully.

  • Check that all links work correctly
  • Ensure images display properly
  • Confirm CSS is linked correctly
  • Check spelling and grammar

Testing is an essential step and should be mentioned when answering theory questions about website development.

12. Common Exam Mistakes to Avoid

  • Missing alternative text for images
  • Incorrect file or folder names
  • Using inline CSS instead of external CSS
  • Poor colour contrast
  • Ignoring client requirements

Avoiding these mistakes can significantly improve exam performance.

13. CSS Properties Commonly Tested in Exams

In Cambridge IGCSE ICT examinations, students are not expected to memorise every CSS property. However, there are several properties that appear frequently in both theory and practical tasks.

  • color – sets the text colour
  • background-color – sets the background colour
  • font-family – changes the font type
  • font-size – changes the size of text
  • text-align – aligns text (left, center, right)
  • padding – space inside an element
  • margin – space outside an element
body {
  font-family: Arial, sans-serif;
  background-color: #f4f6f8;
}

h1 {
  color: navy;
  text-align: center;
}

Using these properties correctly helps create a clear and professional layout and demonstrates good design practice.

14. Why External CSS Is Preferred

External CSS is written in a separate file and linked to the HTML page. Cambridge prefers this method because it clearly separates content from presentation.

Advantages of external CSS include:

  • Consistent design across multiple pages
  • Easier maintenance and editing
  • Cleaner HTML code
  • Better exam practice

Using inline CSS or embedded CSS when external CSS is requested may result in lost marks.

15. Colour Choice and Contrast

Colour choice affects both usability and accessibility. Text should always be clearly visible against its background.

Poor colour contrast makes content difficult to read and can disadvantage users with visual impairments.

Good practice includes:

  • Dark text on a light background
  • Avoiding bright or clashing colours
  • Using consistent colour themes

16. Accessibility in Web Design

Accessibility means designing web pages so that all users, including those with disabilities, can access and understand the content.

Accessibility features that are commonly tested include:

  • Alternative text for images
  • Clear and simple navigation
  • Readable font sizes
  • Logical page structure using headings

In theory questions, students should always mention accessibility when discussing good website design.

17. Target Audience and Purpose

When designing a website, it is important to consider the target audience. The design, language, and layout should suit the intended users.

Examples:

  • A school website should use clear language and formal design
  • A children’s website should use bright colours and simple navigation
  • A business website should appear professional and organised

Cambridge examiners often ask students to explain how a website design is suitable for a particular audience.

18. Examiner Keywords You Should Use

When answering theory questions, using correct terminology helps to gain full marks. Common examiner keywords include:

  • Usability
  • Accessibility
  • Consistency
  • Navigation
  • Target audience
  • Purpose

Including these terms in your answers shows clear understanding of web authoring concepts.

19. Using Tables in Webpages

Tables are used to display data in rows and columns. In Cambridge IGCSE ICT, tables are commonly used for timetables, price lists, and comparison data.

A basic table structure includes the following elements:

  • <table> – defines the table
  • <tr> – table row
  • <th> – table heading
  • <td> – table data
<table>
  <tr>
    <th>Day</th>
    <th>Subject</th>
  </tr>
  <tr>
    <td>Monday</td>
    <td>ICT</td>
  </tr>
</table>

Tables should be used only for displaying data, not for page layout.

20. Introduction to Forms (Theory Level)

Forms are used to collect user input. In IGCSE ICT, students are usually tested on understanding form elements rather than processing data.

Common form elements include:

  • Text boxes
  • Radio buttons
  • Checkboxes
  • Drop-down lists
  • Submit buttons
<form>
  <label>Name:</label>
  <input type="text">

  <input type="submit" value="Send">
</form>

Students should remember that forms in IGCSE ICT are mainly for structure and design, not for advanced scripting.

21. Image File Formats and Optimisation

Choosing the correct image file format is important for quality and file size. Cambridge expects students to understand common image formats.

  • JPEG (.jpg) – best for photographs
  • PNG (.png) – best for logos and transparent images
  • GIF (.gif) – simple animations

Large image files can slow down a website. Images should be resized and compressed before being used on a webpage.

22. Image Size and Proportions

Images should not be stretched or distorted. Changing image size should maintain the original proportions to avoid poor appearance.

Incorrectly resized images are a common practical exam mistake.

23. Validating and Checking HTML

Validation is the process of checking HTML code for errors. Valid code ensures that webpages display correctly across different browsers.

Before submitting work in an exam, students should:

  • Check for missing closing tags
  • Ensure correct nesting of elements
  • Check spelling of file names and links
  • Confirm CSS is linked correctly

Validation helps prevent simple errors that can cost marks.

24. Final Exam Advice for Web Authoring

In practical examinations, always read the question carefully and follow client instructions exactly. Marks are awarded for accuracy rather than creativity.

Key reminders:

  • Use external CSS when required
  • Include alternative text for images
  • Check file names and paths carefully
  • Test all links before submission

Careful checking and calm execution are essential for success in Web Authoring tasks.

25. Summary of Web Authoring Concepts

Web authoring involves designing and creating webpages that are structured, visually clear, accessible, and suitable for their intended purpose. In Cambridge IGCSE ICT, students are expected to demonstrate both theoretical understanding and practical skills.

Throughout this section, you have learned how HTML is used to structure a webpage and how CSS is used to control its appearance. You have also learned the importance of usability, accessibility, and careful testing.

26. Web Authoring Revision Checklist

Before an examination or practical test, use the following checklist to ensure you are fully prepared:

  • I can create a valid HTML page with correct structure
  • I can link an external CSS file correctly
  • I can use headings, paragraphs, lists, and images
  • I can create working hyperlinks and navigation menus
  • I understand relative file paths
  • I include alternative text for all images
  • I choose suitable colours and fonts
  • I test my webpage before submission

27. Common Theory Question Patterns

Cambridge theory questions often ask students to explain or justify design choices. Typical question formats include:

  • Explain why a webpage is suitable for a given audience
  • Describe how accessibility has been considered
  • State reasons for using external CSS
  • Identify improvements to a poorly designed webpage

When answering these questions, always use correct terminology and relate your answers to usability and accessibility.

28. High-Value Exam Keywords

Using appropriate technical terms helps examiners recognise understanding. Important keywords for Web Authoring include:

  • Structure
  • Consistency
  • Navigation
  • Accessibility
  • Usability
  • Target audience
  • Purpose
  • Validation

29. Final Advice for Students

In Web Authoring tasks, accuracy is more important than creativity. Always read instructions carefully and follow them exactly. Small errors such as missing files, incorrect paths, or missing alternative text can result in lost marks.

Stay calm, work methodically, and test your work thoroughly before submission. These habits greatly improve performance in practical examinations.

Practice Pane – Apply & Test Your Skills

Practice Task 1: Basic Webpage Structure

Create a new HTML file and include the correct HTML structure. Your webpage must contain a title, one main heading, and at least one paragraph.

Checklist:

  • DOCTYPE declaration is present
  • HTML, head, and body tags are used correctly
  • Content appears inside the body section

Practice Task 2: Headings and Paragraphs

Add multiple headings and paragraphs to your webpage. Use headings in the correct order and ensure the content is clear and readable.

Exam Tip: Do not skip heading levels unnecessarily.

Practice Task 3: Images and Alternative Text

Insert an image into your webpage. The image must be stored in an images folder and must include appropriate alternative text.

Checklist:

  • Correct image file path is used
  • Image displays correctly
  • Meaningful alt text is included

Practice Task 4: Hyperlinks and Navigation

Create a navigation menu that links to at least two other pages. Ensure all links work correctly.

Common mistake: Incorrect file names or missing extensions.

Practice Task 5: External CSS

Create an external CSS file and apply styles to your webpage. Change the background colour, font, and heading alignment.

Checklist:

  • CSS file is external
  • CSS is linked correctly
  • Styles are applied as intended

Exam-Style Question 1

Explain two reasons why external CSS is preferred when designing a website.

Exam-Style Question 2

Describe two ways a webpage can be made accessible to all users.

Exam-Style Question 3

A student has created a webpage, but the images do not appear. State two possible reasons for this problem.

Self-Assessment Checklist

  • I can create and structure an HTML page correctly
  • I can link and apply external CSS
  • I can use images and links without errors
  • I understand accessibility and usability
  • I test my work before submission

Reflection

After completing these tasks, review your work and identify any areas that need improvement. Practising regularly will build confidence and accuracy.

Mock Practical Test – Web Authoring

This mock test is designed to simulate the style and difficulty of the Cambridge IGCSE ICT practical examination. Students should attempt all tasks under timed conditions.

Scenario

You have been asked to create a small website for a school ICT department. The website must be clear, accessible, and suitable for students and parents.

Task 1: File and Folder Structure (5 marks)

Create a folder structure for the website. The structure must include:

  • An HTML file for the homepage
  • An external CSS file
  • An images folder

Marking guidance:

  • Correct folder structure – 2 marks
  • Correct file naming – 2 marks
  • Logical organisation – 1 mark

Task 2: Page Structure and Content (10 marks)

Create a webpage that includes:

  • A main heading and at least two subheadings
  • Two paragraphs of relevant content
  • A navigation menu linking to two other pages

Marking guidance:

  • Correct HTML structure – 4 marks
  • Appropriate use of headings and paragraphs – 4 marks
  • Working navigation links – 2 marks

Task 3: Images and Accessibility (8 marks)

Insert at least one image related to the school. The image must be stored in the images folder and must include suitable alternative text.

Marking guidance:

  • Correct image insertion – 3 marks
  • Correct file path – 2 marks
  • Meaningful alternative text – 3 marks

Task 4: External CSS Styling (12 marks)

Use an external CSS file to:

  • Set a background colour
  • Change the font family
  • Style headings and navigation links
  • Improve spacing using margins and padding

Marking guidance:

  • Correct linking of CSS file – 3 marks
  • Effective use of CSS properties – 6 marks
  • Clear and consistent design – 3 marks

Task 5: Testing and Validation (5 marks)

Test your website thoroughly. Ensure that all links work, images load correctly, and styles are applied as intended.

Marking guidance:

  • All links function correctly – 2 marks
  • No missing images – 2 marks
  • Clear evidence of testing – 1 mark

Total Marks: 40

Examiner Hints

  • Marks are awarded for accuracy, not creativity
  • Follow instructions exactly
  • Use external CSS where required
  • Check file paths carefully
  • Do not leave tasks incomplete

Self-Evaluation After the Test

After completing the mock test, review your work and answer the following:

  • Which task did I find most difficult?
  • Did I lose marks due to avoidable mistakes?
  • How can I improve my accuracy next time?
