Worked Example: Creating and Evaluating a Report

This step-by-step walkthrough demonstrates how to create and evaluate a report using a realistic dataset — Students by Subject, Grouped by Class with Average Mark.

Step 1 — SQL Query

SELECT ClassName, SubjectName, AVG(Mark) AS AverageMark
FROM StudentMarks
GROUP BY ClassName, SubjectName
ORDER BY ClassName, SubjectName;

This SQL query calculates the average mark per subject for each class. It uses the AVG() aggregate function and groups results by both ClassName and SubjectName.

Step 2 — Query Design Grid

The design grid above shows the equivalent setup in Access: fields ClassName, SubjectName, and Mark with the Totals row set to Group By for the first two and Avg for Mark.

Step 3 — Report Layout View

The layout view arranges the report grouped by Class, then by Subject. The average mark is displayed in each subject group footer. Report headers and footers contain titles and page numbers.

Step 5 — Evaluation Criteria

  • Is grouping logical and clearly shown (Class → Subject)?
  • Are averages correctly formatted to one decimal place?
  • Does the report include a suitable title and page footer?
  • Are all records and subjects represented without omission?
  • Is layout consistent and readable when printed or exported?

Tip: Dear Students,try test reports with varied datasets (e.g., missing values, outliers) to verify design robustness.
