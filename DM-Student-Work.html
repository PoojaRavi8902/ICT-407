Student Step-by-Step Guide — Chapter 18: Data Manipulation

Follow these steps exactly. If you get stuck, re-read the step and try again — do not skip. I will give you the screenshots if required.

Before you start — setup checklist

  • Computer with database software (MS Access, LibreOffice Base or equivalent).
  • Sample database file: Students.accdb (or .odb/.db).
  • Sample CSV file with errors: students_errors.csv.
  • USB or cloud folder to save your work.
Tip: Save a copy of the sample database into your own folder and work on that copy only.

Task 1 — Validation & Integrity (Follow every step)

  1. Open the sample database file
    Open Students.accdb from the folder your teacher gave you. Do not open other files.
  2. Inspect the tables
    Open the Students table in design view and read the field names and data types.
  3. Set validation rules
    For each field, set the following properties (use table design view):
    • StudentID — Data type: Number; Required = Yes
    • FullName — Data type: Text; Required = Yes
    • Phone — Data type: Text; Input mask = 011######### (or numeric type if preferred)
    • Marks — Data type: Number; Validation Rule: >=0 And <=100
    • Class — Lookup: choose from list (Year7, Year8, Year9)
  4. Import the CSV with errors
    Use the Import wizard: choose students_errors.csv. During import, accept suggestions but DO NOT overwrite the table—import into a NEW table called ImportCheck.
  5. Review rejected or flagged rows
    Open ImportCheck and look for fields with missing values, wrong formats or invalid marks (<0 or >100). Note them in your worksheet.
  6. Fix errors using a form
    Open a data-entry form for ImportCheck (or create a simple form). Edit invalid rows so they meet validation rules. Save changes.
  7. Compare counts
    Run a simple query: Count records in ImportCheck before and after cleaning. Note the counts on your worksheet.
  8. Enforce referential integrity (demo)
    Open Relationships → link Students (StudentID) to Enrolments (StudentID). Tick Enforce Referential Integrity. Try to delete a student who has enrolments (teacher demo or show error). Record what happens.
  9. Save your file
    Save the database as YourName_Chapter18.accdb in your folder.
Submit: a screenshot of the import errors before/after and the query counts. Add 2 lines explaining the fixes you made.

Task 2 — Manipulating Data (Sort / Filter / Query)

  1. Create a Select Query
    Open Query Design. Add the Students table. Add fields: StudentID, FullName, Class, Marks.
  2. Filter by marks
    In the Criteria row under Marks, type: >75. Run the query and save it as HighMarks.
  3. Sort results
    Open HighMarks in design view. Under FullName, set Sort = Ascending. Run to check order.
  4. Add a calculated field
    In the query field grid, create a new column: Grade: IIf([Marks]>=90,"A",IIf([Marks]>=75,"B","C")) . Run and check results. Save.
  5. Parameter query (optional stretch)
    Replace the criteria with: [Enter minimum marks:] — run. When prompted, type 80. Observe returned records.
  6. Insert / Update / Delete demo (teacher-led)
    Teacher will demonstrate safe update: show how to edit one record via form (update), add a new student (insert), and create a delete query with a clear WHERE criteria (delete). Students must NOT run delete queries without teacher permission.
Submit: screenshot of HighMarks showing calculated Grade column and a short note: “How did calculated field work?”

Task 3 — Reporting & Exporting

  1. Use the Report Wizard
    Choose HighMarks as source. Select fields: FullName, Class, Marks, Grade.
  2. Group results
    Group by Class and set subtotals: Count of students. Add sort by Marks Descending within group.
  3. Format the report
    Add Title: "High Marks Report". Add page footer with Page Number and Date (use Insert → Page Number / Date).
  4. Preview and export
    Preview the report. Export to PDF: File → Export → PDF. Save as YourName_HighMarks.pdf.
Submit: the exported PDF and one-line explanation: "Who will use this report and why?"

Task 4 — Import / Export & Backup (Short)

  1. Export a table as CSV
    Select table Students → External Data → Export → Text File. Choose CSV and save as YourName_Students.csv.
  2. Re-import into a new table
    Import YourName_Students.csv into the database as Reimport. Verify field types during import wizard.
  3. Create a backup
    File → Save As → Backup Database. Save a copy to your cloud folder or USB with date in filename (e.g., YourName_Chapter18_backup_2025-11-11.accdb).
Important: Keep backups before making large changes. If something goes wrong, you will restore from backup.

Troubleshooting — common problems & quick fixes

  • Import fails because of wrong delimiter: retry import and choose correct delimiter (comma or tab).
  • Calculated field shows #Error: check field names for typos and correct data types.
  • Report shows empty groups: ensure query has data and grouping field is included.
If you cannot fix a problem after 10 minutes, write a short note of what you tried and raise your hand — do not tinker further.
