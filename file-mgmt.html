11.1 Manage files effectively

This section teaches the practical skills required by the syllabus: locate, open, import, save, export, and print files of different types, and to plan a hierarchical folder structure with appropriate file names. Below you'll find detailed step-by-step guides, exam-focused tips and many worked examples.

Locate and open files (Windows / macOS / common tips)

Windows: Use File Explorer (Win + E). Search using the search box and filters (date modified, file type). Use Quick Access and Libraries for frequently used folders.

macOS: Use Finder. Use Spotlight (Cmd + Space) to search by name or content. Use Tags for quick grouping.

Common exam tips: The task may require locating a file by name or extension, or opening it with a specified application. Always show the full file path if asked — e.g., C:\Users\Ravi\Documents\ICT\Report.docx.

Importing files of different types

Importing means bringing outside data into an application. Common imports:

  • Insert an image into a document (.jpg, .png)
  • Import a CSV into a spreadsheet
  • Open an HTML file in a browser (View in browser / View source)
  • Attach or embed a database report

Example (Spreadsheet): Data > Get External Data > From Text/CSV > choose delimiter > finish. Save as .xlsx when completed.

Planning a hierarchical directory / folder structure

Good folder structures balance simplicity and specificity. Use a top-level project folder, then subfolders by document type, date, or student.

ICT_Coursework_2025/
├─ 01_Topics/
│  ├─ Unit1_Types.docx
│  └─ Unit2_Notes.pdf
├─ Assignments/
│  ├─ Assignment1/
│  │   ├─ Draft_Report_v1.docx
│  │   └─ Final_Report_v2.docx
│  └─ Assignment2/
└─ Resources/
   ├─ Images/
   └─ Data/

Exam tip: If asked to create folders in a specific way, follow the instruction exactly — spellings, underscores, and capitals matter in auto-marked tasks.

Saving files with appropriate names

Use descriptive names and versioning. A good pattern: Topic_StudentName_Date_Version.ext

Examples:

  • MarketResearch_Ravi_20250917_v1.docx
  • Sales_Q1_2025_v3.xlsx
  • Website_homepage_2025-09-17.html

Avoid spaces if the exam environment is strict; use underscores or hyphens. Do not use illegal characters like / \ : * ? " < > |.

Printing and screenshots

Printing tasks may require you to print documents, graphs or reports. Screenshots are often required as evidence. Typical steps:

  1. Arrange content to fit a page (margins, page orientation)
  2. File > Print > choose printer or 'Save as PDF'
  3. Take screenshots: Windows (PrtScr / Snipping Tool), macOS (Cmd+Shift+4)

Exam tip: Save screenshots in a separate evidence folder named clearly (e.g., ExamEvidence_Shots) and include the time stamp if required.

File formats — characteristics and uses

This section covers both application-specific formats (e.g., .docx, .xlsx, .accdb) and generic formats (.csv, .txt, .pdf, .jpg/.png, .html). Understanding when to use each format and the consequences (size, editability, compatibility) is essential for exams.

Saving & Exporting — practical step-by-step

Many exam tasks ask you to save or export in a specific format. Follow these sample procedures exactly (exam environments often check file names and locations).

Save a document in different formats (.docx, .rtf, .pdf)
  1. File > Save As > choose folder > enter file name (follow naming rules).
  2. Choose the required format from the dropdown (e.g., Word Document (.docx)).
  3. To create a PDF: File > Export > Create PDF/XPS > Save (or Print > Save as PDF).
  4. Double-check by opening the saved file: open the .pdf in a PDF viewer to confirm layout.
Export spreadsheet as .csv
  1. Open the spreadsheet, ensure the sheet you want is active.
  2. File > Save As > choose CSV (Comma delimited) (*.csv).
  3. Note: Only the active sheet will be saved in CSV. Formulas are converted to values.
  4. Open the CSV in a text editor to confirm comma separation.
Save a web page in browser & HTML view
  1. Browser view: File > Save Page As > Webpage, Complete > This creates an HTML file plus a resource folder.
  2. HTML view: Right-click > View Page Source (or use a code editor). Save the HTML file if required with extension .htm or .html.
  3. To submit source code, the examiner may ask for the HTML file — ensure the file shows the correct extension.
Exporting database reports

Database packages often allow report export to PDF, RTF or CSV. Steps (example): Reports > Select report > Export > Choose format (PDF/RTF/CSV) > Save.

Check exported report for layout and field truncation. If a report is too wide, change orientation to landscape before exporting.

11.2 Reduce file sizes for storage or transmission (Compression)

Compression reduces file sizes so files are easier to store or send. There are two types of compression:

  • Lossy compression — reduces size by discarding some data (e.g., JPG for photos, MP3 for audio).
  • Lossless compression — reduces size without losing data (e.g., ZIP, PNG for images).
How to compress files (.zip / .rar)
  1. Select files/folders > Right-click > Send to > Compressed (zipped) folder (Windows) or Compress "Folder" (macOS).
  2. Use archive utilities (7-Zip, WinRAR) if available to choose compression level and format (.zip or .rar).
  3. To open: double-click or use Extract > choose destination.

Exam tip: If asked to create a .zip containing specific files, include only the requested files and name the archive exactly as instructed (e.g., Submission_Ravi.zip).

Why compress? (practical reasons)
  • Smaller files are faster to upload and download.
  • Some email systems limit attachment size; compression helps to stay within limits.
  • Packaging multiple files into a single archive simplifies submission.
When compression is NOT appropriate
  • Already-compressed formats (e.g., JPEG) may not shrink much further.
  • When the receiving system cannot extract archives — always check exam instructions.
  • For final publishing, PDFs and images may be better left unzipped.

Practice Tasks & Exam-style Questions

Below are many practice tasks that mirror the types of activities you will be asked to perform in the IGCSE ICT practical exam.

Task A — Folder structure & file naming (5–8 marks)

  1. Create a folder called IGCSE_ICT_2025 and inside it create subfolders Notes, Assignments and Evidence.
  2. Save a Word document named Notes_Ravi_20250917.docx into Notes.
  3. Export the Word document to PDF and save it in Evidence with the name Notes_Ravi_20250917.pdf.

Mark scheme hints: Folder names must match exactly, files must be placed in correct subfolders and the exported PDF must open and show the same content.

Task B — Export & CSV (6–10 marks)

  1. Open the provided spreadsheet and export the active worksheet as students_data.csv.
  2. Open students_data.csv in a text editor and copy the first 10 lines into a new text file called sample.txt.

Hints: Ensure the CSV uses commas as separators. If the spreadsheet contains commas inside cells, export using quotation marks around text fields.

Task C — Compression & naming (4–7 marks)

  1. Compress the folder Assignment2 into Assignment2_Ravi.zip.
  2. Open the archive and confirm that FinalReport_v1.docx is present.

Hints: The zip should contain the folder structure — not separate files flattened unless specified.

Worked example — Exporting a database report to PDF

Steps (example using Access): Open the report > File > Save as > PDF or XPS > Choose file name > Save. Verify the PDF by opening it and checking column headings and page breaks.

Quick Reference Cheat-sheet

  • Naming: Topic_Name_YYYYMMDD_v1.ext
  • Common formats: .docx, .xlsx, .pptx, .pdf, .csv, .jpg, .png, .html, .zip
  • Compress: Right-click > Send to > Compressed (zipped) folder
  • Save as PDF: File > Export > Create PDF/XPS
  • CSV notes: Only active sheet saved; formulas become values
  • HTML submission: save .html and include any local resource files in same folder
