21.1 Web development layers

Content layer (HTML) — provides the structure and semantic meaning of content. Examples: <h1>, <p>, <table>, <img>.

Presentation layer (CSS) — controls layout and style: fonts, colours, spacing, responsive behaviour. Can be internal (style tag), external (linked stylesheet), or inline (style attribute).

Behaviour layer (JavaScript) — adds interactivity: form validation, collapsible menus, live preview, DOM manipulation. In exams you must explain the role (not necessarily write complex JS).

Exam-style explanation (short answer):

Content (HTML) should be separate from presentation (CSS) and behaviour (JavaScript). Separating layers helps maintenance, accessibility, and accessibility marks in the exam.

Click to toggle a minimal three-layer example
<!-- index.html (Content) -->
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <title>Three-layer demo</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <h1>Hello</h1>
  <button id="sayHi">Say hi</button>
  <script src="app.js"></script>
</body>
</html>

/* styles.css (Presentation) */
body{font-family:Arial}
h1{color:darkblue}

// app.js (Behaviour)
document.getElementById('sayHi').addEventListener('click', ()=>alert('Hi'));

21.2 Create a web page — head and body

Head section: required elements and common meta tags

  • <title> — required for pages and exam answers.
  • <meta charset="utf-8"> — prevents character encoding issues.
  • <meta name="viewport" content="width=device-width, initial-scale=1"> — mobile responsiveness.
  • Link external stylesheet(s) using <link rel="stylesheet" href="...">. Use the correct file path and order: default browser styles < user stylesheet < inline styles (last priority).
  • Optional: <meta name="description">, keywords, author, and favicons.

Body section: content elements

The <body> contains the visible content: headings, paragraphs, lists, images, tables, multimedia, forms, links, and anchors. In exam answers, show correct nesting and closing tags.

Head + Body minimal template (exam-ready) 
<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>IGCSE Web Authoring Sample</title>
  <link rel="stylesheet" href="styles/site.css">
</head>
<body>
  <header><h1>Web Authoring</h1></header>
  <main><p>Content goes here</p></main>
  <footer><p>Author: Teacher</p></footer>
</body>
</html>

Head ordering rules (important for marks)

  1. Place <meta charset> before any text content.
  2. Include responsive viewport meta before CSS that relies on it.
  3. External CSS links should go in the <head> so content appears styled without flicker.
Exam tip: When asked to create a page in the exam, always include the full document structure (doctype, html, head, meta tags, title, body). Missing <title> often loses easy marks.

HTML elements used in Web Authoring

Lists (ordered and unordered)

Use ordered lists (<ol>) for sequences and unordered lists (<ul>) for bulleted items. Use <li> inside both.

<h2>Steps to publish</h2>
<ol>
  <li>Create files in correct folder</li>
  <li>Test links locally</li>
  <li>Upload via FTP</li>
</ol>

<h2>Topics</h2>
<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>

Tables (including caption, thead/tbody, and colspan/rowspan)

Tables are common in exam questions. Use semantic structure (<caption>, <thead>, <tbody>) and include colspan or rowspan where asked.

<table>
  <caption>Class list</caption>
  <thead>
    <tr><th>ID</th><th>Name</th><th>Role</th></tr>
  </thead>
  <tbody>
    <tr><td>1</td><td>Aisha</td><td>Student</td></tr>
    <tr><td colspan="2">Totals</td><td>30</td></tr>
  </tbody>
</table>

Common table markup errors

  • Missing closing tags for <tr> or <td>.
  • Using border attribute instead of CSS (use CSS in modern pages).
  • Not using caption or header rows where required.

Images and multimedia

Use <img src="..." alt="..."> and provide meaningful alt text. For audio and video, use <audio> and <video> with controls.

<img src="images/school.jpg" alt="Front of school building" width="600">

<audio controls>
  <source src="audio/speech.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>

<video controls width="640">
  <source src="videos/demo.mp4" type="video/mp4">
  Your browser does not support the video element.
</video>

Hyperlinks and anchors (bookmarks)

Types of links:

  • Internal links — link to other pages within the same site using relative paths: <a href="about.html">About</a>.
  • External links — absolute URLs: <a href="https://www.cambridgeinternational.org">Cambridge</a>.
  • Email links — <a href="mailto:teacher@example.com">Email</a>.
  • Anchors/bookmarks — use id attributes and link with #id to navigate inside a page.
<!-- anchor target -->
<h2 id="publishing">Publishing</h2>
<p>Details on how to publish the site.</p>

<!-- link to anchor -->
<a href="#publishing">Go to Publishing section</a>

<!-- relative link -->
<a href="resources/index.html">Resources</a>

<!-- external open in new tab (use target) -->
<a href="https://example.org" target="_blank" rel="noopener">External site</a>
Remember: When opening external links in a new tab, include rel="noopener" for security — examiners may award marks for security awareness.

21.3 Use stylesheets (Presentation layer)

Stylesheets control the visual presentation. Key concepts examiners test:

  • Types of CSS: External (<link>), Internal (<style>), Inline (style="...").
  • Selector types: element, class (.class), id (#id), descendant, child, attribute selectors.
  • Box model: content, padding, border, margin.
  • Layout: normal flow, floats (legacy), flexbox (recommended), and simple responsive rules using media queries.
  • Specificity & cascade: inline > id > class > element; later rules override earlier ones when specificity is equal.
/* external styles.css */
body{font-family:Arial, sans-serif;line-height:1.5}
.container{max-width:1000px;margin:0 auto;padding:10px}
h1{font-size:28px}
.nav .active{font-weight:bold}

/* responsive example */
@media (max-width:600px){
  .grid{grid-template-columns:1fr}
}

Exam-style short question

Explain the difference between class selectors and id selectors. (2 marks)

Common CSS mistakes

  • Overusing inline styles (reduces maintainability).
  • Wrong file path to external CSS in <link> tag.
  • Forgetting vendor prefixes is usually not required in exams, but know why they exist.

21.4 Test and publish a website

Testing checklist

  • Functionality: all links work, forms submit (or show expected behaviour), multimedia plays.
  • Cross-browser check: pages render in common browsers (Edge, Chrome, Firefox).
  • Responsive behaviour: use devtools to check small screens.
  • Accessibility checks: alt text present, headings in logical order, link text meaningful.
  • Performance: optimize images, use correct file formats and sizes.

Publishing methods

Common exam expectation: show understanding of FTP and relative/absolute paths.

  1. Prepare file structure (index.html at root, assets in folders such as /css, /images, /js).
  2. Use an FTP client (e.g., FileZilla) or deploy via simple hosting services (GitHub Pages, Netlify) — explain the steps in a short answer.
  3. Check permissions and that default files (index.html) load from the server root.
Exam short answer (example):

To publish via FTP: open the FTP client, connect using host/user/password, navigate to the public_html (or www) folder on the server, upload the website folder contents keeping relative paths intact. Test the site by opening the domain in a browser.
