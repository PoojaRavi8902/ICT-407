4.1.1 Network Devices

1. Modem

Definition
A modem (short for modulator-demodulator) is a device that converts digital signals from a computer into analogue signals that can travel through telephone lines — and vice versa.
Detailed Explanation
  • Computers work with digital signals (1s and 0s).
  • Traditional telephone lines carry analogue signals.
  • A modem modulates digital data into analogue form for transmission and demodulates it back to digital form on receipt.

Two common types:

  1. Dial-up Modem – Old technology, very slow (≈60 Kbps).
  2. Broadband/ADSL Modem – Modern modems using existing phone lines or fibre optic cables.
    • “Asymmetric” means download speed > upload speed.
    • ADSL modems can remain always connected and still allow telephone use.
Example
When a home computer connects to the internet via a telephone line, the modem converts computer data to analogue form for transmission and converts incoming analogue data back into digital signals.

2. Hub

Definition
A hub is a hardware device that connects multiple computers or devices in a Local Area Network (LAN) and sends incoming data to all devices connected to it.
Detailed Explanation
  • When a data packet reaches one port of the hub, it is broadcast to every connected device.
  • This means all devices receive the data — even those for whom it isn’t intended.
  • This process causes network congestion and security risks.

Hubs are now mostly replaced by switches, which are faster and more efficient.

Example
In a small office, five computers connected to a hub will each receive every message sent, even if the message is meant for just one of them.
Figure: How a Hub Works — every connected device receives all packets sent through the hub.

3. Switch

Definition
A switch is a network device that connects multiple computers or devices and sends data only to the intended recipient by using the device’s MAC address.
Detailed Explanation
  • Switches improve network efficiency and security.
  • Each connected device has a unique MAC address stored in the switch’s table.
  • The switch checks the data packet’s destination MAC address and forwards it only to that specific device.
  • This reduces unnecessary traffic and improves performance.
Example
In a classroom LAN, when Computer A sends data to Computer B, only B receives it — not the others.
Figure: How a Switch Works — packets only go to the correct destination.

4. Bridge

Definition
A bridge is a network device that connects two or more LANs using the same protocol, enabling them to function as a single network.
Detailed Explanation
  • Used to join sections of a LAN together (for example, connecting two floors of a school).
  • Bridges control data traffic between networks, ensuring packets go only where needed.
  • Prevents network flooding caused by unnecessary data broadcasting.
Example
Connecting the computer lab LAN and the library LAN within a school building using a bridge.

5. Router

Definition
A router is a device that forwards data packets between different networks, such as between a LAN and the Internet (WAN).
Detailed Explanation
  • Routers determine the best path for data packets to reach their destination.
  • They use routing tables containing IP addresses and MAC addresses to direct packets.
  • Home routers often combine the roles of router, switch, firewall, and wireless access point.
Example
Your home Wi-Fi router connects your personal devices (LAN) to your Internet Service Provider’s network (WAN).

6. Gateway

Definition
A gateway acts as a doorway between networks, enabling data to flow between networks that use different protocols.
Detailed Explanation
  • All networks have boundaries; communication outside requires a gateway.
  • Converts data from one format/protocol to another (e.g. company intranet to Internet).
Example
When an internal email is sent to someone outside the company, the data passes through a gateway that adapts the internal format to Internet-compatible protocols.

7. Network Interface Card (NIC)

Definition
A NIC is a hardware component that allows a computer or device to connect to a network.
Detailed Explanation
  • Contains the device’s unique MAC address.
  • Can be wired (Ethernet port) or wireless (Wi-Fi adapter).
  • Enables data transmission and reception within a network.
Example
Every laptop or desktop has a NIC — either as a built-in component or as a plug-in USB Wi-Fi adapter.

8. Network Cables

Detailed Explanation

Although wireless is common, wired connections are still widely used because they:

  • Provide faster data transfer rates.
  • Are generally more secure.

Cable types:

  • Copper cables (Ethernet) – Cheaper, adequate for short distances.
  • Fibre optic cables – Faster, more secure, supports very high bandwidth.

4.1.2 Internet Protocol (IP) and Media Access Control (MAC) Addresses

Definition

IP address: A unique numeric address assigned to a device on a network so that other devices can locate and communicate with it.
MAC address: A unique hardware-level address embedded in a network interface (NIC) that identifies a specific device on a network.

Detailed Explanation

IP Addresses

  • Internet Protocol (IP) addresses are used at the network layer (in networking models) to identify where a device is on a network.
  • In IPv4, an IP address is a 32-bit number, often written in “dotted decimal” form, for example: 109.108.158.1.
  • When your home computer connects to the internet, your ISP (Internet Service Provider) assigns you an IP address (sometimes dynamically, sometimes statically).
  • Web servers (e.g. for websites) typically have fixed (static) IP addresses so people can reliably reach them.
  • You can use an IP address in place of a domain name; for example, typing http://109.108.158.1 in a browser may take you to the web server at that address (if that server is hosting a site).

Key points:

  • Two devices cannot share the same IP address (on a given network).
  • IP addresses may change (for dynamic assignment) when devices reconnect or after a lease expires.

MAC Addresses

  • MAC stands for Media Access Control.
  • A MAC address is hard-wired (burned into the NIC by the manufacturer) and is intended to be unique worldwide (for that hardware device).
  • A MAC address is 48 bits long (6 bytes) and typically expressed in hexadecimal notation, for example: 00:1A:2B:3C:4D:5E.
  • The MAC address works at the data link layer, and is used to identify which physical device on a local network (LAN) should receive data packets.
Analogy / Relationship between IP & MAC
  • IP address = where the device is located (like the address or location of a house).
  • MAC address = who the device is, uniquely (like each person living in that house).
  • If you move to a new house (change your IP), your name (MAC address) remains the same.
  • The network uses the IP to route packets to the correct network; once packets arrive in that network, the MAC address is used to deliver them to the exact device.
Comparison: IP Address vs MAC Address
FeatureIP AddressMAC Address
LayerNetwork layerData link layer
PurposeIdentifies location / network addressIdentifies specific hardware device
ChangeableYes (dynamic or static)No (fixed)
Format32-bit (IPv4) or 128-bit (IPv6)48-bit, typically 6 hexadecimal bytes
Used byRouters, switchesSwitches, NICs
Examples
  1. Suppose your home router assigns your laptop the IP 192.168.1.10. The NIC’s MAC might be f4:5c:89:ab:cd:ef. When sending a packet to your printer in the same LAN, your computer uses the printer’s IP to know which network, then the MAC to send to the correct device.
  2. A web server might have IP 203.0.113.25 (static). Many devices around the globe can connect to it using that IP or a domain name mapped to it.

4.1.3 Wi-Fi and Bluetooth

Definitions
  • Wi-Fi: A wireless networking technology that allows devices to connect to a network (typically LAN / Internet) using radio waves.
  • Bluetooth: A wireless technology for short-range data exchange between devices over short distances (often < 30 m) using radio waves and frequency-hopping techniques.

Detailed Explanation

Wi-Fi

  • Wi-Fi stands for “Wireless Fidelity”.
  • A Wireless Access Point (WAP / AP) receives data from the wired network (e.g., Ethernet) and transmits it as radio waves.
  • A Wi-Fi enabled device (laptop, smartphone) has a wireless adapter that receives those waves and converts them to digital data (and vice-versa).
  • Wi-Fi is appropriate for larger networks (homes, schools) — typical range ≈ 100 m (obstacles reduce range).
  • Common encryption standards: WEP, WPA, WPA2, WPA3.

Bluetooth

  • Bluetooth operates in the 2.45 GHz band and uses many channels (classic mode: 79 channels).
  • It uses spread-spectrum frequency hopping — devices rapidly switch channels to avoid interference.
  • Ideal for short-range, low-bandwidth tasks (e.g., headsets, small file transfers).
  • Forms a Wireless Personal Area Network (WPAN) — centred on a user’s devices.
  • Devices pair using key-matching encryption for a secure link.

4.1.4 How to Set Up and Configure a Small Network

Definition / Purpose
Practical steps, hardware and software needed to build a small network (e.g., a school lab of 10 computers) and ensure correct operation, security and maintainability.

Detailed Explanation

1. Hardware Requirements

  • Modem (if internet is required).
  • Router (routes traffic between LAN and Internet).
  • Switches / hubs to connect devices within the LAN.
  • Network Interface Cards (NICs) for each device.
  • Cables (Ethernet / fibre) or wireless APs for Wi-Fi.
  • Firewall (hardware or software) to protect from unauthorised access.

2. Software Requirements / Configuration

  • Network drivers for NICs.
  • IP addressing (static addresses or DHCP server).
  • Server software for file sharing and authentication.
  • Firewall rules and user access privileges.
  • Wireless configuration: SSID, encryption, passwords.

3. Steps to Configure (practical checklist)

  1. Physically connect devices using cables or configure Wi-Fi adapters.
  2. Connect modem → router → switch → client computers.
  3. Enable DHCP on the router (or assign static IPs as needed).
  4. Configure firewall rules and security settings.
  5. Install and configure file server, printers and shared resources.
  6. Create user accounts and set permissions.
  7. Test connectivity: ping devices, open shared folders, browse web.

4. Additional Considerations

  • Network licences: ensure legal use of software across networked machines.
  • Set appropriate user privileges to protect sensitive data.
  • Wireless coverage: plan AP placement to avoid dead zones.
  • Backup & maintenance routines to protect important data and ensure availability.
Figure: Typical small network layout — Internet → Modem → Router → Switch → Server/PCs
Example Scenario
  1. Purchase router, a 16-port switch, Ethernet cables and a Wi-Fi AP.
  2. Connect modem → router → switch → each computer to the switch.
  3. Enable DHCP so each PC receives an IP automatically.
  4. Install file server, create shared folders, set user permissions and install antivirus/firewall.
  5. Test by copying files and browsing the internet from each PC.

  4. Why must you purchase a network license for software used by multiple computers?
  5. After setup, how can you test whether the network is functioning properly?

4.1.5 Internet, Intranets and Extranets

Definitions
  • Internet: A global public network connecting millions of computers using common protocols (TCP/IP).
  • Intranet: A private network within an organisation (school/company) using internet technologies, accessible only to internal users.
  • Extranet: A controlled extension of an intranet allowing limited external access (e.g., partners) via secure connections.
Detailed Explanation
  • Internet is open and global — websites, email and cloud services are available to authorised users worldwide.
  • Intranets are private: used for internal documentation, staff directories, internal communication and shared databases.
  • Extranets allow selected external parties (suppliers, partners) to access certain intranet resources securely (e.g., via VPN).
  • Organisations keep sensitive data on intranets; extranets permit controlled collaboration with external entities.
Examples
  • Internet: Students using YouTube and global email services.
  • Intranet: School portal where teachers upload assignments and timetables.
  • Extranet: Supplier portal where vendors log in to view purchase orders.

4.1.6 LAN, WAN and WLAN

Definitions
  • LAN (Local Area Network): A network covering a small area (building or campus).
  • WAN (Wide Area Network): A network covering large geographic areas (cities, countries), often linking LANs.
  • WLAN (Wireless LAN): A LAN that uses wireless technologies (radio or infrared) instead of cables.

Detailed Explanation

LAN
  • Confined to a single building or campus; devices connect via switches, APs or hubs.
  • Advantages: resource sharing, central management, easy communication.
  • Disadvantages: single server failure can disrupt network; security and virus spread are risks.
WAN
  • Links LANs across large distances using public infrastructure (telephone, satellite) or leased lines.
  • Examples: the global Internet, bank ATM networks.
  • Because of distance, latency is higher and costs are usually greater.
WLAN
  • Wireless equivalent of a LAN using APs and Wi-Fi adapters.
  • Advantages: flexibility, roaming, easier device addition, reduced cabling costs.
  • Disadvantages: security challenges, interference, often slower than wired LAN.

4.1.7 Accessing the Internet

Different devices can be used to access the Internet, such as mobile phones, tablets, laptops, and desktop computers. Each device has its own advantages and disadvantages depending on portability, screen size, cost, and usability.

Mobile Phones and Tablets

  • Advantages: Very portable, ideal for quick browsing, social media, and communication.
  • Disadvantages: Small keyboards, not all websites are mobile-friendly, small display, expensive data without Wi-Fi.

Laptops

  • Advantages: Larger screens, full keyboards, suitable for multimedia and general browsing.
  • Disadvantages: Heavier, may need dongles for mobile networks, slower than desktops.

Desktop Computers

  • Advantages: Powerful hardware, large display, stable wired connection.
  • Disadvantages: Not portable, occupies more space, needs accessories for mobility.

4.2.1 Network Security

Network security involves protecting data and systems from hacking, phishing, and viruses when using the Internet.

Should the Internet Be Policed?

Arguments For

  • Prevents illegal or harmful content.
  • Protects children and vulnerable users.
  • Reduces misinformation and fake news.

Arguments Against

  • Global policing is expensive and difficult.
  • Restricts freedom of information.
  • Laws already exist to handle illegal activity.

The Digital Divide

The digital divide is the gap between those who have access to technology and the Internet and those who do not. It can be caused by lack of money, skills, or infrastructure.

4.2.2 Authentication

Authentication verifies that users or data come from a trusted source, ensuring that only authorised individuals can access systems or data.

  • Usernames and Passwords
  • Digital Certificates
  • Biometrics (fingerprints, facial recognition)
  • Magnetic Stripe and Contactless Cards
  • Holograms and Photo ID Cards (e.g., passports)
