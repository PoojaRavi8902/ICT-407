1. Introduction to Web Authoring

1.1 What is Web Authoring?

Web authoring is the process of planning, designing, creating, and maintaining websites. It involves using specialised languages and tools to produce web pages that can be viewed using a web browser. In the Cambridge IGCSE ICT (0983) syllabus, web authoring focuses specifically on the use of HTML (HyperText Markup Language) and CSS (Cascading Style Sheets).

Web authoring is not programming. HTML and CSS do not perform calculations or logic. Instead, they describe:

  • What content appears on a webpage
  • How that content is structured
  • How that content is presented to the user

In the IGCSE examination, students are assessed on their ability to create web pages that are:

  • Technically correct
  • Well-structured
  • Visually clear
  • Accessible to users
  • Fit for a stated purpose and target audience

1.2 Why Web Authoring is Assessed in IGCSE ICT

Web authoring is included in the IGCSE ICT syllabus because it tests a wide range of ICT skills, including:

  • Understanding of file management and folder structures
  • Ability to follow detailed instructions accurately
  • Application of design principles
  • Awareness of usability and accessibility issues
  • Attention to detail and problem-solving

Unlike theory-only topics, web authoring allows candidates to demonstrate practical competence. This is why it forms a significant part of Paper 2 (Practical Examination).

1.3 How Web Authoring is Assessed

Web authoring skills are assessed in two main ways:

  1. Theory questions – where candidates explain concepts such as usability, accessibility, or the purpose of HTML and CSS.
  2. Practical tasks – where candidates must create, edit, or debug web pages using HTML and CSS.

In the practical exam, candidates are typically given:

  • A scenario (for example, a school website or event website)
  • A set of instructions
  • Existing files and folders
  • Images and resources

Marks are awarded for following instructions precisely. Extra features that are not requested do not gain extra marks.

1.4 Key Web Authoring Technologies in the Syllabus

HTML (HyperText Markup Language)

HTML is used to create the structure of a webpage. It uses elements (also called tags) to define headings, paragraphs, images, links, tables, and forms.

HTML tells the browser:

  • This is a heading
  • This is a paragraph
  • This is an image
  • This is a link

CSS (Cascading Style Sheets)

CSS is used to control the presentation of a webpage. It defines colours, fonts, spacing, layout, and positioning.

CSS allows designers to separate content from presentation, which makes websites:

  • Easier to update
  • More consistent
  • More professional

1.5 Websites, Web Pages, and Home Pages

It is important to understand the difference between key terms:

  • Website – a collection of related web pages stored together
  • Web page – a single HTML document
  • Home page – the main or starting page of a website, usually named index.html

In exams, incorrect use of terminology can lead to lost marks. Always use terms precisely.

1.6 Examiner Advice (Very Important)

According to examiner reports, common weaknesses include:

  • Confusing HTML with CSS
  • Using incorrect tags
  • Ignoring accessibility
  • Not following file structure instructions

Successful candidates:

  • Use correct HTML structure
  • Link external CSS correctly
  • Test their work carefully
  • Read every instruction slowly and carefully

2. Website Purpose, Target Audience and Design Considerations

2.1 Purpose of a Website

Every website is created for a specific purpose. In the Cambridge IGCSE ICT 0983 syllabus, students are expected to understand how the purpose of a website affects its design, content, and structure.

The purpose of a website explains why the website exists. A website that does not clearly meet its purpose will be considered poorly designed, even if it is technically correct.

Common Website Purposes

Purpose Description Example
Informational Provides information to users School or government website
Educational Teaches or trains users Online learning platform
Commercial Sells products or services Online shop
Promotional Advertises an event or organisation Event website
Entertainment Amuses or engages users Game or media website

In the examination, candidates are often given a scenario that clearly states the purpose of the website. All design decisions must support that purpose.

2.2 Target Audience

The target audience is the group of people the website is designed for. Identifying the target audience is essential because different users have different needs, abilities, and expectations.

A website that is suitable for one audience may be completely unsuitable for another. Examiners expect candidates to recognise this and design accordingly.

Examples of Target Audiences

  • Children
  • Teenagers
  • Adults
  • Elderly users
  • Professionals
  • General public

2.3 How Target Audience Affects Website Design

When designing a website, the following features are directly influenced by the target audience:

Language and Tone

Websites for children should use simple language and short sentences. Websites for professionals may use formal language and technical terms.

Font Size and Type

Elderly users often require:

  • Larger font sizes
  • Clear, simple fonts
  • High contrast between text and background

Teenagers may prefer modern fonts and visually engaging layouts.

Colour Choices

Colour choices must be appropriate for both the audience and the purpose. Bright colours may be suitable for children, but not for a professional business website.

Navigation Style

Younger or less experienced users benefit from:

  • Simple navigation menus
  • Clear labels
  • Fewer menu options

Complex navigation structures can confuse users and reduce usability.

2.4 Design Considerations in Web Authoring

Design considerations are the factors a web author must think about before and during the creation of a website.

Consistency

Consistency means that the website looks and behaves in the same way across all pages. This includes:

  • Navigation menus in the same position
  • Same colour scheme
  • Same font styles

Consistency improves usability and reduces confusion.

Readability

Readability refers to how easy it is to read text on a website. Factors that affect readability include:

  • Font size
  • Font type
  • Line spacing
  • Colour contrast

White Space

White space is the empty space between elements on a webpage. It helps to:

  • Separate content
  • Improve readability
  • Make pages look less crowded

2.5 Usability and User Experience

Usability measures how easy a website is to use. A usable website allows users to complete tasks quickly and without frustration.

In the IGCSE exam, candidates are often asked to explain how design choices improve usability.

Features of a Usable Website

  • Clear navigation
  • Logical layout
  • Readable text
  • Consistent design

2.6 Accessibility Considerations

Accessibility ensures that websites can be used by people with disabilities. This is a core theory requirement in the IGCSE ICT syllabus.

Common Accessibility Features

  • Alt text for images
  • Labels for form fields
  • High contrast colour schemes
  • Simple and clear language

In exam questions, candidates must link accessibility features to specific user needs.

2.7 Exam-Style Questions

Short Answer Question

Question: State one way the target audience affects website design.

Model Answer: The target audience affects the choice of font size and colour to ensure the website is easy to read.

Structured Question

Question: Explain two ways a website for elderly users should be designed.

Model Answer:
1. Large font sizes should be used so text is easy to read.
2. High contrast colours should be used to improve visibility.

2.8 Common Examiner Comments

Examiner reports frequently mention that candidates:

  • Describe design features without linking them to purpose or audience
  • Give generic answers that lack explanation
  • Forget to mention accessibility

High-scoring answers always:

  • Refer to the target audience
  • Link design choices to user needs
  • Use correct terminology

3. HTML Fundamentals: Structure, Tags, Syntax and Examiner Pitfalls

3.1 What is HTML?

HTML stands for HyperText Markup Language. It is the standard language used to create web pages. HTML is not a programming language; it is a markup language.

A markup language uses tags to describe the structure and meaning of content. HTML tells a web browser how different parts of a webpage should be displayed, but it does not control detailed appearance such as colours or fonts.

In Cambridge IGCSE ICT 0983, candidates are expected to:

  • Use correct HTML syntax
  • Choose appropriate tags for content
  • Structure web pages logically
  • Avoid deprecated or incorrect tags

3.2 HTML Tags and Elements

An HTML tag is a keyword enclosed in angle brackets, for example <p>. An HTML element usually consists of:

  • An opening tag
  • Content
  • A closing tag

Example: 

<p>This is a paragraph.</p>

Some elements are empty elements and do not have a closing tag.

Example: 

<img src="photo.jpg" alt="A photograph">

3.3 Basic HTML Document Structure

Every HTML document should follow a standard structure. Failure to include this structure often results in lost marks in the practical exam. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>
    <h1>Main Heading</h1>
    <p>Main content goes here.</p>
</body>
</html>

Explanation of Each Part

Tag Purpose
<!DOCTYPE html> Declares the document as HTML5
<html> Root element of the webpage
<head> Contains metadata and settings
<title> Text shown in browser tab
<body> Visible page content

3.4 The <head> Section in Detail

The <head> section contains information about the webpage that is not displayed directly to the user.

Common elements inside the <head> section include:

  • <title>
  • <meta>
  • <link> (for CSS)

Content such as paragraphs, images, and headings should never be placed inside the <head> section.

3.5 The <body> Section

The <body> section contains all content that users see when viewing the webpage in a browser.

This includes:

  • Text
  • Images
  • Links
  • Lists
  • Tables
  • Forms

In the practical exam, most marks are awarded for work done inside the <body> section.

3.6 Headings in HTML

HTML provides six levels of headings: 

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Section Heading</h3>
<h4>Subsection Heading</h4>
<h5>Smaller Heading</h5>
<h6>Smallest Heading</h6>

Headings should be used in a logical hierarchy. The <h1> tag should be used for the main heading of the page, followed by <h2> and so on.

Skipping heading levels (for example, using <h1> followed by <h4>) is poor practice and may affect accessibility.

3.7 Paragraphs and Line Breaks

Paragraphs are created using the <p> tag. 

<p>This is a paragraph of text.</p>

A line break can be created using the <br> tag. This should be used sparingly. 

First line<br>
Second line

3.8 Emphasis and Text Formatting

HTML provides tags for emphasising text:

  • <strong> – strong importance
  • <em> – emphasis
  • <b> – bold (presentation only)
  • <i> – italic (presentation only)

In IGCSE ICT, <strong> and <em> are preferred because they have semantic meaning.

3.9 HTML Comments

Comments are used to explain code and are not displayed in the browser. 

<!-- This is a comment -->

Comments can help examiners understand your work, especially in complex tasks.

3.10 Common HTML Errors (Examiner Reports)

  • Missing closing tags
  • Incorrect nesting of elements
  • Content placed outside the <body>
  • Incorrect use of headings
  • Spelling mistakes in tag names

3.11 Exam-Style Questions

Multiple Choice

Which tag contains all visible content on a webpage?

A. <head>
B. <html>
C. <body>
D. <title>

Correct answer: C

Structured Question

Question: Explain two reasons why correct HTML structure is important.

Model Answer:
1. It ensures the webpage displays correctly in different browsers.
2. It improves accessibility and makes the code easier to maintain.

4. Text, Images, Hyperlinks, Lists and Tables

4.1 Presenting Text on a Web Page

Text is the most common type of content on a webpage. In Cambridge IGCSE ICT 0983, candidates are expected to use appropriate HTML tags to present text clearly and logically.

Correct text presentation improves:

  • Readability
  • Usability
  • Accessibility
  • Professional appearance

Paragraphs

Paragraphs must be created using the <p> tag. Each paragraph should represent a separate idea or topic. 

<p>This is a paragraph of text.</p>

Using multiple paragraphs instead of long blocks of text makes content easier to read and understand.

Line Breaks

The <br> tag inserts a line break without starting a new paragraph. It should only be used where a line break is logically required, such as in addresses. 

123 High Street<br>
London<br>
UK

4.2 Images in Web Pages

Images are used to enhance visual appeal and support textual content. In the IGCSE practical exam, images must be inserted correctly and stored in the correct folder.

The <img> Tag

<img src="images/school.jpg" alt="Front of the school building">

The <img> tag is an empty element and does not have a closing tag.

Image Attributes

Attribute Purpose
src Specifies the location of the image file
alt Provides alternative text for accessibility
width / height Controls image size (better done with CSS)

The alt attribute is compulsory in the IGCSE exam. Missing alt text is a very common reason for losing marks.

Image File Formats

Format Features Typical Use
JPEG (JPG) Small file size, lossy compression Photographs
PNG High quality, supports transparency Logos, icons
GIF Limited colours, supports animation Simple animations

4.3 File Paths for Images (Very Important)

In the practical exam, images are usually stored in a separate folder. Correct use of relative file paths is essential. 

<img src="images/logo.png" alt="Company logo">

Common errors include:

  • Incorrect folder name
  • Missing file extension
  • Incorrect use of upper and lower case

4.4 Hyperlinks (Links)

Hyperlinks allow users to navigate between web pages or visit external websites. They are created using the <a> tag.

Internal Links

<a href="about.html">About Us</a>

Internal links connect pages within the same website. All internal links must be tested before submission.

External Links

<a href="https://www.example.com">Visit Example</a>

External links connect to websites outside your own site.

Email Links

<a href="mailto:info@school.org">Email Us</a>

Opening Links in a New Tab

<a href="https://www.example.com" target="_blank">Visit Website</a>

Opening links in a new tab is usually appropriate for external websites.

4.5 Lists

Lists are used to organise information clearly. There are two main types of lists in HTML.

Unordered Lists

<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>Accessibility</li>
</ul>

Ordered Lists

<ol>
    <li>Plan the website</li>
    <li>Create HTML</li>
    <li>Apply CSS</li>
</ol>

Nested Lists

<ul>
    <li>Frontend
        <ul>
            <li>HTML</li>
            <li>CSS</li>
        </ul>
    </li>
</ul>

Nested lists are commonly used in navigation menus and structured content.

4.6 Tables

Tables are used to display structured data. They must not be used for page layout.

Basic Table Structure

<table>
    <tr>
        <th>Day</th>
        <th>Subject</th>
    </tr>
    <tr>
        <td>Monday</td>
        <td>ICT</td>
    </tr>
</table>

Table Tags Explained

Tag Purpose
<table> Creates the table
<tr> Table row
<th> Header cell
<td> Data cell

4.7 Accessibility Considerations

All content elements must be accessible. This includes:

  • Alt text for images
  • Clear link text (avoid “click here”)
  • Proper table headers

4.8 Common Examiner Errors

  • Missing alt text
  • Broken links
  • Tables used for layout
  • Incorrect file paths

4.9 Exam-Style Questions

Multiple Choice

Which attribute is required to improve image accessibility?

A. src
B. width
C. alt
D. height

Correct answer: C

Structured Question

Question: Explain two reasons why tables should not be used for page layout.

Model Answer:
1. Tables make code harder to read and maintain.
2. Tables reduce accessibility and flexibility of layout design.

5. File Organisation, Folder Structure and Relative Paths

5.1 Importance of File Organisation in Web Authoring

File organisation refers to how files and folders are arranged within a website project. In Cambridge IGCSE ICT 0983, correct file organisation is essential because it directly affects whether images, stylesheets, and links work correctly.

Examiner reports consistently highlight that many candidates lose marks due to poor file organisation rather than poor HTML or CSS knowledge.

Good file organisation:

  • Prevents broken links and missing images
  • Makes websites easier to update and maintain
  • Allows examiners to clearly understand your work
  • Ensures portability of the website between computers

5.2 Typical Website Folder Structure

In the practical exam, candidates are usually expected to use a clear and logical folder structure. A typical structure is shown below. 

website/
│
├── index.html
├── about.html
├── contact.html
│
├── css/
│   └── style.css
│
└── images/
    ├── logo.png
    └── banner.jpg

This structure separates different types of files:

  • HTML files in the root folder
  • CSS files in a css folder
  • Image files in an images folder

5.3 Naming Files and Folders

File and folder names must be chosen carefully. Incorrect naming is a common cause of broken links.

Rules for File Naming

  • Use lowercase letters
  • Avoid spaces (use hyphens if needed)
  • Use meaningful names
  • Include correct file extensions

Good and Bad Examples

Good Practice Poor Practice
index.html Home Page.html
about.html AboutPage.HTML
style.css Style Sheet.css

Some computer systems treat uppercase and lowercase letters as different. This means Home.html and home.html are not the same file.

5.4 Relative File Paths

A relative file path describes the location of a file relative to the current file. Relative paths are required in the IGCSE exam.

Linking a CSS File

<link rel="stylesheet" href="css/style.css">

Linking an Image

<img src="images/logo.png" alt="Company logo">

These examples assume:

  • The HTML file is in the root folder
  • The CSS file is in the css folder
  • The image file is in the images folder

5.5 Linking Between Pages Using Relative Paths

Internal hyperlinks must use relative paths to link between pages in the same website. 

<a href="about.html">About Us</a>

If pages are stored in subfolders, the path must reflect this accurately.

Using Parent Folder Paths

The .. symbol is used to move up one folder level. 

<a href="../index.html">Home</a>

This is often required when linking from a page inside a subfolder.

5.6 Absolute Paths (Why They Are Not Used)

An absolute path shows the full location of a file, including the drive letter or full web address. 

C:/Users/Student/Desktop/website/images/logo.png

Absolute paths must not be used in the practical exam because:

  • They only work on one computer
  • They break when files are moved
  • They are not portable

Relative paths ensure the website works correctly on any computer.

5.7 Common File Organisation Errors (Examiner Reports)

  • Images stored in the wrong folder
  • Incorrect file extensions
  • Broken internal links
  • Incorrect relative paths
  • Using absolute paths

5.8 Practical Exam Tips

  • Create folders before writing code
  • Check file names carefully
  • Test every link and image
  • Save work frequently

5.9 Exam-Style Questions

Short Answer

Question: State one advantage of using relative file paths.

Model Answer: Relative paths allow the website to work correctly when moved to another computer.

Structured Question

Question: Explain two problems caused by poor file organisation in a website.

Model Answer:
1. Images may not display due to incorrect file paths.
2. Links between pages may not work, reducing usability.

6. Forms: Input Types, Labels, Validation and Examiner Pitfalls

6.1 Purpose of Forms in Web Authoring

Forms are used to collect data from users. In the Cambridge IGCSE ICT 0983 syllabus, candidates are not required to process or store form data using databases or scripting. Instead, the focus is on creating forms that are:

  • Well-structured
  • Easy to use
  • Accessible
  • Fit for the given scenario

Forms are commonly used for:

  • Contact pages
  • Registration forms
  • Feedback surveys
  • Event sign-up pages

6.2 Basic Form Structure

All form elements must be placed inside a <form> tag. 

<form action="#" method="post">
    <!-- form elements go here -->
</form>

Form Attributes Explained

Attribute Purpose
action Specifies where the form data is sent
method Specifies how data is sent (get or post)

In the IGCSE exam, action="#" is acceptable unless otherwise stated in the question.

6.3 Text Input Fields

Text input fields are used to collect short pieces of text. 

<input type="text" name="fullname">

Common uses include:

  • Name
  • Username
  • City

6.4 Password Input

Password inputs hide the characters typed by the user. 

<input type="password" name="password">

This improves privacy but does not encrypt data.

6.5 Email Input

The email input type automatically checks for a valid email format. 

<input type="email" name="email">

This is preferred over text inputs when collecting email addresses.

6.6 Number Input

<input type="number" name="age">

This restricts input to numbers only.

6.7 Radio Buttons

Radio buttons are used when the user must select one option only. 

<input type="radio" name="gender" value="male"> Male
<input type="radio" name="gender" value="female"> Female

All radio buttons in the same group must share the name attribute.

6.8 Checkboxes

Checkboxes allow users to select multiple options. 

<input type="checkbox" name="subject" value="ict"> ICT
<input type="checkbox" name="subject" value="maths"> Maths

6.9 Drop-Down Lists (Select Menus)

<select name="country">
    <option>Norway</option>
    <option>UK</option>
    <option>India</option>
</select>

Drop-down lists save space and reduce input errors.

6.10 Text Areas

Text areas are used for long text input. 

<textarea name="message"></textarea>

6.11 Labels and Accessibility

Labels are essential for accessibility and usability. Every input field should have a label. 

<label for="email">Email Address:</label>
<input type="email" id="email">

Labels allow screen readers to correctly describe form fields.

6.12 Required Fields

The required attribute prevents form submission unless the field is completed. 

<input type="text" required>

6.13 Placeholder Text

<input type="email" placeholder="Enter your email">

Placeholder text provides guidance but does not replace labels.

6.14 Grouping Form Elements

The <fieldset> tag groups related form elements. 

<fieldset>
    <legend>Personal Details</legend>
    <input type="text">
</fieldset>

6.15 Submit and Reset Buttons

<input type="submit" value="Submit">
<input type="reset" value="Clear">

6.16 Common Examiner Errors (Forms)

  • Missing labels
  • Incorrect input types
  • Radio buttons with different names
  • Missing required attributes

6.17 Exam-Style Questions

Multiple Choice

Which input type ensures that only one option can be selected?

A. checkbox
B. text
C. radio
D. number

Correct answer: C

Structured Question

Question: Explain two reasons why labels are important in forms.

Model Answer:
1. Labels improve accessibility for screen reader users.
2. Labels make forms easier to understand and use.

7. CSS Fundamentals: Purpose, Selectors, Properties and Syntax

7.1 What is CSS?

CSS stands for Cascading Style Sheets. CSS is used to control the presentation of web pages. While HTML defines the structure and content of a webpage, CSS defines how that content looks.

In Cambridge IGCSE ICT 0983, CSS is used to:

  • Control colours
  • Control fonts and text size
  • Control spacing and alignment
  • Create consistent design across multiple pages

A key principle examined in IGCSE ICT is the separation of content and presentation. HTML should be used only for structure, while CSS should be used only for styling.

7.2 Advantages of Using CSS

CSS offers several important advantages over using HTML tags or attributes for styling.

  • Improves consistency across pages
  • Makes websites easier to update
  • Reduces repetition of code
  • Improves accessibility
  • Produces a more professional appearance

In theory questions, candidates are often asked to explain why CSS should be used instead of HTML. Answers must refer to maintenance, consistency, or separation of content and presentation.

7.3 Ways of Applying CSS

There are three ways to apply CSS to a webpage. All three may appear in exam questions, but one is clearly preferred.

7.3.1 Inline CSS

Inline CSS is written directly inside an HTML tag. 

<p style="color: red;">This is red text.</p>

Disadvantages of inline CSS:

  • Not reusable
  • Difficult to maintain
  • Mixes content and presentation

7.3.2 Internal CSS

Internal CSS is written inside a <style> tag in the <head> section of the HTML document. 

<head>
    <style>
        p {
            color: blue;
        }
    </style>
</head>

Internal CSS applies to the entire page, but not to multiple pages.

7.3.3 External CSS (Best Practice)

External CSS is written in a separate file with a .css extension. This is the preferred method in the IGCSE practical exam. 

<link rel="stylesheet" href="css/style.css">

Advantages of external CSS:

  • One file controls many pages
  • Consistent design
  • Easier to maintain
  • Cleaner HTML code

7.4 CSS Syntax

A CSS rule consists of a selector and a declaration block. 

selector {
    property: value;
}

Example: 

p {
    color: black;
    font-size: 14px;
}

7.5 CSS Selectors

CSS selectors determine which HTML elements a style rule applies to. In IGCSE ICT, three main types of selectors must be understood.

7.5.1 Element Selector

p {
    font-size: 16px;
}

This applies the style to all paragraph elements.

7.5.2 Class Selector

.highlight {
    background-color: yellow;
}
    
<p class="highlight">Important text</p>

Class selectors are reusable and are commonly used in the exam.

7.5.3 ID Selector

#header {
    background-color: lightblue;
}
    
<div id="header">Header content</div>

ID selectors must be unique on a page.

7.6 Common CSS Properties

Text Colour

color: #000000;

Background Colour

background-color: #f4f4f4;

Font Family

font-family: Arial, sans-serif;

Font Size

font-size: 16px;

7.7 CSS Comments

/* This is a CSS comment */

Comments help explain code and improve readability.

7.8 Common CSS Errors (Examiner Reports)

  • Forgetting to link the external CSS file
  • Missing semicolons
  • Incorrect selectors
  • Using inline CSS instead of external CSS

7.9 Exam-Style Questions

Multiple Choice

Which type of CSS is preferred in the IGCSE practical exam?

A. Inline
B. Internal
C. External
D. Embedded

Correct answer: C

Structured Question

Question: Explain two advantages of using external CSS.

Model Answer:
1. External CSS allows the same styles to be applied to multiple pages, ensuring consistency.
2. External CSS makes websites easier to maintain because changes only need to be made in one file.

8. CSS Box Model, Page Layout Techniques and Structure

8.1 Understanding Page Layout in Web Design

Page layout refers to how content is arranged on a web page. In Cambridge IGCSE ICT 0983, candidates are expected to create layouts that are clear, logical, and suitable for the given purpose and target audience.

Layout is assessed mainly in the practical exam and is closely linked to:

  • Usability
  • Readability
  • Accessibility
  • Professional appearance

8.2 The CSS Box Model (Very Important)

In CSS, every HTML element is treated as a rectangular box. This concept is known as the CSS Box Model. Understanding the box model is essential for controlling spacing and layout.

Components of the Box Model

  1. Content – the actual text or image
  2. Padding – space between the content and the border
  3. Border – the line surrounding the padding and content
  4. Margin – space outside the border

Visual Representation (Conceptual)

Margin
-------------------------
|       Border          |
|  -------------------  |
|  |     Padding     |  |
|  |  ------------   |  |
|  |  |  Content |   |  |
|  |  ------------   |  |
|  |                |  |
|  -------------------  |
-------------------------

8.3 Padding

Padding controls the space inside an element, between the content and the border. 

p {
    padding: 10px;
}

Padding increases the clickable area of elements and improves readability.

8.4 Margin

Margin controls the space outside an element, separating it from other elements. 

div {
    margin: 20px;
}

Margins are commonly used to separate sections of a page.

8.5 Borders

Borders surround the padding and content of an element. 

div {
    border: 2px solid black;
}

Borders are often used to visually group content.

8.6 Width and Height

The width and height properties control the size of elements. 

img {
    width: 300px;
    height: auto;
}

Using height: auto helps maintain the correct image proportions.

8.7 Page Layout Using Div Elements

The <div> element is a generic container used to group content for styling and layout.

A simple page layout often includes:

  • Header
  • Navigation
  • Main content area
  • Footer

Basic Layout Structure (HTML)

<div id="header">Header</div>
<div id="nav">Navigation</div>
<div id="content">Main content</div>
<div id="footer">Footer</div>

Basic Layout Styling (CSS)

#header, #nav, #content, #footer {
    padding: 15px;
    margin-bottom: 10px;
}

8.8 Fixed-Width and Fluid Layouts

Fixed-Width Layout

A fixed-width layout uses a specific width value. 

#content {
    width: 960px;
    margin: auto;
}

Fixed layouts are easy to control but do not adapt to screen size.

Fluid Layout

A fluid layout uses percentage widths. 

#content {
    width: 80%;
    margin: auto;
}

Fluid layouts adjust to different screen sizes.

8.9 Clearing Common Layout Misconceptions

  • Tables should not be used for layout
  • Spacing should be controlled with CSS, not HTML
  • Layout should support content, not distract from it

8.10 Examiner-Reported Layout Errors

  • Overcrowded pages
  • No spacing between sections
  • Inconsistent layout across pages
  • Poor alignment of elements

8.11 Exam-Style Questions

Multiple Choice

Which CSS property controls space outside an element?

A. padding
B. margin
C. border
D. width

Correct answer: B

Structured Question

Question: Explain the difference between margin and padding.

Model Answer:
Margin controls the space outside an element, separating it from other elements, while padding controls the space inside an element between the content and the border.

9. Navigation Menus, Usability and Accessibility

9.1 Purpose of Navigation in a Website

Navigation refers to the system of links that allows users to move between pages in a website. In Cambridge IGCSE ICT 0983, navigation is a critical feature because it directly affects usability.

A website with poor navigation is difficult to use, even if the content itself is good. Examiners expect navigation to be:

  • Clear
  • Consistent
  • Easy to understand
  • Present on all pages

9.2 Types of Navigation

Text-Based Navigation

Text-based navigation is the most common type used in the IGCSE exam. It is simple, accessible, and easy to implement. 

<nav>
    <a href="index.html">Home</a> |
    <a href="about.html">About</a> |
    <a href="contact.html">Contact</a>
</nav>

Text-based navigation works well for all users and devices.

List-Based Navigation

Navigation menus are often created using unordered lists. 

<ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="contact.html">Contact</a></li>
</ul>

This structure is especially suitable for styling with CSS.

9.3 Navigation Consistency

Consistent navigation means that the navigation menu:

  • Appears in the same position on every page
  • Uses the same wording on every page
  • Links to the same destinations

In the practical exam, marks are often lost when:

  • Navigation is missing on one or more pages
  • Links point to incorrect files
  • Different pages use different navigation menus

9.4 Usability

Usability measures how easy a website is to use. A usable website allows users to complete tasks quickly and without confusion.

In theory questions, candidates are frequently asked to explain how specific design choices improve usability.

Key Usability Features

  • Clear navigation
  • Logical layout
  • Readable text
  • Consistent design
  • Clear feedback to users

9.5 Accessibility

Accessibility ensures that websites can be used by people with disabilities. This is a core theory requirement in Cambridge IGCSE ICT.

Candidates must be able to:

  • Identify accessibility features
  • Explain how they help specific users

Accessibility Features and Their Benefits

Feature Who It Helps Why
Alt text Visually impaired users Screen readers can describe images
Labels Screen reader users Inputs are clearly identified
High contrast colours Users with poor vision Text is easier to read
Simple language Cognitive disabilities Content is easier to understand

9.6 Designing for Accessibility in Navigation

Navigation should be:

  • Easy to read
  • Easy to click
  • Clearly labelled

Avoid vague link text such as Click here. Instead, use meaningful link text such as Contact Us.

9.7 Common Examiner Errors

  • Broken navigation links
  • Navigation missing on some pages
  • Poor contrast in navigation text
  • Overcrowded menus

9.8 Exam-Style Questions

Short Answer

Question: State one reason why consistent navigation is important.

Model Answer: Consistent navigation helps users find information easily on every page.

Structured Question

Question: Explain two ways a website can be made accessible for visually impaired users.

Model Answer:
1. Alt text can be added to images so screen readers can describe them.
2. High contrast colours can be used to make text easier to read.

10. Testing, Debugging and Common Examiner Errors

10.1 Importance of Testing in Web Authoring

Testing is the process of checking that a website works correctly before it is published or submitted. In Cambridge IGCSE ICT 0983, testing is essential because marks are only awarded for work that functions correctly.

A website may look correct in the code editor but still contain errors that prevent it from working as intended. Candidates who do not test their work often lose marks unnecessarily.

Testing should be carried out:

  • After completing each task
  • After making changes to code
  • Before submitting the final work

10.2 What Should Be Tested?

In the practical exam, candidates should test all aspects of the website, including:

  • All internal and external links
  • All images
  • CSS styling
  • Forms and validation
  • Page layout and alignment

10.3 Testing Hyperlinks

All hyperlinks must be clicked to ensure they lead to the correct destination. Broken links are one of the most common reasons for lost marks.

Common causes of broken links include:

  • Incorrect file names
  • Incorrect file paths
  • Missing file extensions
  • Incorrect use of upper and lower case

Example of an Incorrect Link

<a href="Home.html">Home</a>

Corrected Version

<a href="home.html">Home</a>

10.4 Testing Images

Images should be checked to ensure they:

  • Display correctly
  • Use the correct file path
  • Have appropriate alt text
  • Are sized correctly

Common Image Errors

  • Image stored in the wrong folder
  • Incorrect file extension
  • Missing alt attribute
  • Distorted image due to incorrect sizing

10.5 Testing CSS

CSS should be tested to ensure that styles are applied as intended. A very common mistake is forgetting to link the external CSS file.

Checking the CSS Link

<link rel="stylesheet" href="css/style.css">

If CSS is not working, candidates should check:

  • The file path to the CSS file
  • The file name and extension
  • Spelling of selectors
  • Missing semicolons or brackets

10.6 Debugging HTML Errors

Debugging is the process of finding and fixing errors in code. HTML errors often cause content to display incorrectly or not at all.

Missing Closing Tags

<p>This is a paragraph

Correct version: 

<p>This is a paragraph</p>

Incorrect Nesting

<p><h1>Title</h1></p>

Correct version: 

<h1>Title</h1>
<p>Text</p>

10.7 Debugging CSS Errors

Missing Semicolon

p {
    color: red
    background-color: blue;
}

Correct version: 

p {
    color: red;
    background-color: blue;
}

Incorrect Selector

header {
    background-color: grey;
}

If the HTML uses id="header", the correct selector is: 

#header {
    background-color: grey;
}

10.8 Testing Forms

Forms must be tested to ensure that:

  • All required fields prevent submission when empty
  • Radio buttons behave correctly
  • Checkboxes allow multiple selections
  • Email fields reject invalid input

10.9 Examiner-Reported Mistakes

According to examiner reports, common reasons for losing marks include:

  • Not testing work before submission
  • Ignoring error messages
  • Leaving broken links unfixed
  • Failing to check form validation

10.10 Practical Exam Testing Checklist

  • All pages open correctly
  • All links work
  • All images display
  • CSS is applied correctly
  • Forms behave as expected
  • No visible error messages

10.11 Exam-Style Questions

Short Answer

Question: State one reason why testing is important in web authoring.

Model Answer: Testing ensures that all parts of the website work correctly and no marks are lost due to errors.

Structured Question

Question: Explain two problems caused by not testing a website before submission.

Model Answer:
1. Broken links or missing images may not be noticed, resulting in lost marks.
2. Form validation errors may prevent users from submitting data correctly.
