4.1.4 How to Set Up and Configure a Small Network

Definition / Purpose
This section explores the practical steps, hardware, and software considerations to build a small network (for example, a school lab of 10 computers) and ensure it works smoothly.

Detailed Explanation
To set up a small network, you must consider:

  1. Hardware Requirements
    • A modem (if connecting to the internet).
    • A router (to route traffic between the LAN and Internet).
    • Switches/hubs to connect all devices within the LAN.
    • Network Interface Cards (NICs) in each device (if not built-in).
    • Cables (Ethernet, fibre, etc.) if wiring the network.
    • Wireless components (WAPs) if Wi‑Fi is needed.
    • Firewall (hardware or software) for security.
  2. Software Requirements / Configuration
    • Network drivers for NICs.
    • IP address assignments (static or via DHCP).
    • Network operating system or server software (for file sharing, authentication).
    • Security setup (firewall, access privileges).
    • Wireless network configuration (SSID, passwords, encryption).
  3. Steps to Configure
    1. Physically connect devices via cables or Wi‑Fi.
    2. Connect your router to the modem (for internet).
    3. Connect switches/hubs to the router and devices to the switches.
    4. Assign IP addresses (or enable DHCP).
    5. Configure firewall rules.
    6. Set up shared file servers, printers, etc.
    7. Define user accounts and permissions (who can access what).
    8. Test connectivity (ping devices, test internet, file sharing).
  4. Additional Considerations
    • Network licences: ensure legal license for networked software use.
    • Privileges / security: limit user access appropriately.
    • Wireless coverage: ensure enough WAPs to avoid dead zones.
    • Backup & maintenance: regularly backup and monitor network health.

Example Scenario
You have 10 computers in a lab needing internet and file sharing: purchase a router, 16‑port switch, Ethernet cables and a Wi‑Fi AP; connect modem→router→switch→PCs; enable DHCP; install file server; set accounts; test connectivity.

4.1.5 Internet, Intranets and Extranets

Definitions

  • Internet: global public network using TCP/IP.
  • Intranet: private network within an organization using internet technologies for internal access only.
  • Extranet: extension of an intranet allowing limited external access (partners) via secure connections.

Intranets are closed; extranets give controlled external access (e.g., supplier portals via VPN).

4.1.6 LAN, WAN and WLAN

Definitions

  • LAN: Local Area Network — small geographic area (building/campus).
  • WAN: Wide Area Network — large geographic area (cities, countries).
  • WLAN: Wireless LAN — uses radio / Wi‑Fi instead of cables.

Comparisons & Examples — LAN: lab network; WAN: internet or inter-branch communication; WLAN: same lab with Wi‑Fi APs. Consider security, cost and speed differences.

Phase 2 — Short Lecture (10 minutes)

Teaching notes (10 minutes)

  1. Introduce hardware & software needed for a small network — modem, router, switch, NIC, cables, WAPs, firewall.
  2. Demonstrate physical layout: modem → router → switch → PCs / WAPs.
  3. Explain DHCP vs static IPs and why DHCP simplifies lab setup.
  4. Discuss intranet vs internet vs extranet with school examples.
  5. Summarise LAN/WAN/WLAN differences and practical uses.

Tip: draw the simple network on the board and show a router admin page if available.
