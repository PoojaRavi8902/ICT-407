How exam questions usually “feel”

Cambridge-style questions often ask you to explain, compare, and justify. That means you must do more than name a device or topology. You need to connect the idea to a reason:

Compare: “A LAN covers a small area and is usually privately owned; a WAN covers a large area and uses leased lines or public infrastructure.”
Justify: “A switch is preferred over a hub because it forwards frames only to the destination port, reducing collisions and improving security.”
Explain impacts: “Networks enable collaboration and centralised backups, but create security risks such as malware spreading quickly.”

In this platform you’ll repeatedly practise those sentence patterns so your exam answers sound “mark-scheme accurate”.

Exam habit that wins marks: Use because / therefore / so to connect cause → effect. Example: “Star topology is easy to manage because each node has its own cable to the switch, so a broken cable affects only one device.”

4.1 Types of Networks — LAN, WAN, WLAN (Wi-Fi)

A network is a group of devices connected so they can share data and resources. In exams, the most common trap is giving vague definitions. Avoid “LAN is small, WAN is big.” Instead, include: area covered, ownership, typical speed, and examples.

LAN — Local Area Network

A LAN connects computers and devices within a small geographical area such as a classroom, a school building, an office floor, or a home. A LAN is usually owned and managed by one organisation or person. Because it covers a small area, it can use high-speed cabling (Ethernet) and modern Wi-Fi, so LANs are typically fast.

LAN characteristics:
• Small area (room/building/campus)
• Usually privately owned
• High speed / low delay
• Often uses Ethernet + switches; may use Wi-Fi access points
LAN exam mistake:
Don’t say “LAN uses Wi-Fi” as if Wi-Fi = LAN. Wi-Fi is just a wireless method. A LAN can be wired or wireless.
Real-world picture: school LAN

In a school, computers in classrooms connect to switches. Those switches connect to a main switch and a router. Printers and shared folders are on the LAN. Students log in and access files stored on a server. The LAN may also connect to Wi-Fi access points so laptops and tablets can join.

Advantages (explain “why”)

Fast data sharing because devices are close and can use high-speed links. Resource sharing (printers, storage) because the network allows multiple users to access the same device. Centralised management

Disadvantages (explain impacts)

Setup cost (cabling, switches, access points). Maintenance (updates, troubleshooting). Security risk because if malware enters the LAN it can spread quickly to shared folders and other PCs.

WAN — Wide Area Network

A WAN connects devices or entire LANs across a large geographical area such as multiple cities, countries, or continents. WAN links are often provided through telecommunication companies (leased lines, fibre backbones, cellular networks, satellite links). WANs tend to be slower than LANs because distances are large and more networking equipment is involved.

WAN characteristics:
• Very large area (country/continent)
• Uses public/leased infrastructure
• Higher cost than LAN
• Often connects multiple LANs together
Exam phrase you should use:
“A WAN typically connects LANs over long distances using telecom infrastructure.”
Example: a company with branches

A company might have a headquarters LAN in Cairo and a branch office LAN in Alexandria. A WAN link connects the two, so staff can access shared databases, email systems, and internal services. Without a WAN, each branch becomes isolated.

Why is the Internet called a WAN?

The Internet is the largest WAN because it connects millions of networks across the world. It uses many technologies (fibre, submarine cables, satellites, routers) to move data between networks.

Advantages and disadvantages (with “because”)

Advantage: Global communication and remote access because users can connect to systems from different locations.
Disadvantage: Security threats because data travels through more public infrastructure and has more exposure to attacks.
Disadvantage: Higher cost and lower speed because long-distance links require telecom services and more routing equipment.

WLAN — Wireless LAN (Wi-Fi)

A WLAN is a LAN where devices connect using wireless radio signals (Wi-Fi), usually through an access point or wireless router. WLANs are popular because they allow movement: laptops, phones, tablets.

Strengths

Mobility, easy installation, fewer cables, flexible layouts. Useful where running cables is difficult.

Weaknesses

Security risk if encryption/passwords are weak; interference from walls or other networks; limited range.

Exam trap: WLAN ≠ WAN. WLAN is still local; it’s the wireless method that changes.

LAN in one perfect sentence: A LAN connects devices within a small area (e.g., school/office), is usually privately owned, and offers high-speed sharing of files and peripherals.

Think “inside one site”. LANs are about controlled, fast local sharing. A LAN can use wired Ethernet or Wi-Fi.

Think “between sites”. WANs connect LANs. Speeds may be lower and delays higher compared to LAN.

WLAN in one perfect sentence: A WLAN is a local network that uses Wi-Fi radio signals via an access point to connect devices without cables, providing mobility but needing strong security and good coverage.

Think “LAN without wires”. WLAN is about convenience and movement, but it needs careful security (e.g., WPA2/WPA3).

LAN vs WAN (2–4 marks)

A LAN covers a small area such as a building and is usually privately owned, whereas a WAN covers a large area and connects networks using telecom infrastructure. LANs tend to have higher speeds and lower delays because devices are close together, while WANs are often slower and more expensive because they use long-distance links.

LAN vs WLAN (2–4 marks)

Both are local networks, but a WLAN uses wireless radio signals (Wi-Fi) through an access point, while a LAN can be wired. WLAN offers mobility and easier installation but can suffer interference and needs strong security due to eavesdropping risks.

4.2 Network Hardware — Router, Switch, Hub, NIC, Modem

In exams, network hardware questions reward clear “job descriptions”. If you can describe what each device does in one correct sentence and compare similar devices (router vs switch, hub vs switch), you gain marks quickly.

Router — “connects networks”

A router connects different networks together and decides where data should go next. Most home routers connect your home LAN to your Internet Service Provider (ISP). Routers forward packets using IP addresses.

Key phrase: A router connects LANs/WANs and forwards packets using IP addresses.
Router in a home network

Your devices (phones, laptops) are on a private LAN (often 192.168.x.x). The router sends traffic to the ISP and brings Internet responses back. Many routers also include Wi-Fi access point features.

Switch — “connects devices inside a LAN”

A switch connects devices within a LAN and sends data only to the correct device. It learns which device is on which port by reading MAC addresses.

Key phrase: A switch forwards frames to the correct port using MAC addresses.
Why switches reduce collisions

With a hub, all devices share the same communication channel and collisions happen easily. With a switch, each link is dedicated; the switch directs data to only one port, so fewer collisions occur.

Hub — “broadcasts to all”

A hub is an older device that connects devices in a LAN but broadcasts data to every port. Even if the data is meant for one device, all devices receive it.

Exam line: A hub is less secure because all devices can see the traffic.
Hub vs switch (most asked comparison)

A hub broadcasts to all ports, increasing collisions and reducing security. A switch forwards to the correct port, improving efficiency and security.

NIC — Network Interface Card

A NIC is the hardware that allows a device to connect to a network (wired or wireless). Each NIC has a unique MAC address used in local network communication.

Key phrase: NIC provides network connectivity and has a unique MAC address.
Why MAC addresses matter

Switches use MAC addresses to learn where devices are connected. MAC addresses are usually fixed for the NIC, unlike IP addresses which can change.

Modem — the bridge to the ISP

A modem connects your network to the ISP’s system. The word comes from MOdulate and DEModulate: it converts signals so that data can travel over the ISP’s medium (copper line, cable, fibre, etc.).

Key phrase: A modem converts signals and links to the ISP.
Common confusion: In many homes, the “router box” includes both modem and router functions.

4.3 Network Topologies — Star, Bus, Ring, Mesh

A topology is how devices are arranged and connected. Exam questions love topologies because they test both knowledge and reasoning: cost, reliability, and fault-finding.

Your best strategy: learn each topology using the same four headings: (1) Structure, (2) Advantage, (3) Disadvantage, (4) Best use case.

Star topology

In a star topology, every device connects to a central device (usually a switch). Data passes through the central device to reach the destination.

Advantages:
• Easy to add/remove devices
• Easy to fault-find (one cable affects one device)
• Good performance with a switch
Disadvantages:
• Central device failure stops the network
• More cable needed than bus

Bus topology

In a bus topology, all devices share a single backbone cable. Signals travel along the backbone and devices pick up messages addressed to them.

Advantages:
• Cheap, less cable
• Simple to install for small networks
Disadvantages:
• Backbone failure stops whole network
• Collisions likely under heavy use
• Harder to troubleshoot

Ring topology

In a ring topology, devices connect in a circle. Data travels around the ring (often one direction) until it reaches the destination.

Advantages:
• No collisions in controlled token systems
• Predictable access
Disadvantages:
• A break can affect the whole network
• Adding/removing nodes can be disruptive

Mesh topology

In a mesh topology, devices have multiple connections to other devices. A full mesh connects every device to every other; a partial mesh connects some.

Advantages:
• Very reliable
• No single point of failure
• Multiple routes for data
Disadvantages:
• Expensive (many cables/ports)
• Complex to install and manage

Why star is common in schools/offices (6–8 marks)

Star topology is common because it is easy to manage and expand: each device has its own cable to a central switch, so adding a new computer simply means connecting it to a spare port. Fault-finding is easier because if one cable fails, only that device loses connection. Performance is good because a switch forwards data only to the intended device, reducing collisions. However, the network depends on the central switch: if it fails, the whole network can stop.

Bus vs mesh (reliability vs cost)

A bus topology is cheap because it uses less cabling, but it is less reliable because the backbone cable is a single point of failure: if it breaks, all devices lose connection. A mesh topology is very reliable because devices have multiple connections and alternative routes; however it is expensive and complex because it needs many cables and ports.

4.4 Internet Concepts — IP, MAC, DNS, Cloud Computing

This section often appears in Paper 1 as short definitions and also as “explain how it works” questions. The top scoring answers show you understand the roles: IP for routing across networks, MAC for local delivery, DNS for name→IP conversion, and cloud as online services.

IP Address (Internet Protocol address)

An IP address identifies a device on a network so data can be delivered to it. Routers use IP addresses to move data across networks (routing).

IPv4 vs IPv6

IPv4 is a 32-bit address (e.g., 192.168.1.25). The number of IPv4 addresses is limited. IPv6 is longer (128-bit) and provides many more addresses, supporting the growth of devices.

Public vs private IP

A private IP is used inside a home/school LAN and is not directly reachable from the Internet. A public IP is used on the Internet and is assigned by an ISP.

Exam sentence: IP addresses help route packets between networks; they can change (e.g., via DHCP).

MAC Address (Media Access Control)

A MAC address is a unique hardware address assigned to a NIC. It is mainly used for communication within a local network (LAN). Switches use MAC addresses to forward frames to the correct port.

Difference to remember:
IP = “where to route across networks” (can change)
MAC = “who the device is locally” (linked to NIC)

DNS (Domain Name System) — how names become numbers

People prefer names like www.cambridgeinternational.org. Computers need numbers (IP addresses). DNS is the system that translates a domain name into its IP address.

DNS steps (exam-friendly)

1) User types a domain name into the browser.
2) The computer asks a DNS server for the IP address of that name.
3) The DNS server replies with the IP address (or passes request to other DNS servers).
4) The browser connects to the web server using the IP address.

Cloud Computing — using services over the Internet

Cloud computing means using computing services (storage, software, processing) over the Internet. Instead of storing everything on your own device, you store data and run apps on remote servers (“the cloud”).

Types of cloud (as asked in ICT)

Public cloud: services provided over the Internet for many users (e.g., common cloud storage providers).
Private cloud: cloud infrastructure used by one organisation only (more control).
Hybrid cloud: mixture of public and private (some sensitive data stays private, other services public).

Advantages vs disadvantages (with reasons)

Advantages: Access anywhere, automatic backups, scalable storage, collaboration.
Disadvantages: Depends on Internet connection, possible privacy/security risks, service outages, subscription costs.

Cloud computing: many devices access the same online services from anywhere.

4.5 Benefits and Drawbacks of Networks

In ICT, “effects” means real outcomes. You should be able to describe benefits and drawbacks in practical contexts: schools, offices, banks, hospitals. High marks come from linking to examples and explaining consequences.

Benefits (what networks enable)

  • Resource sharing: printers, scanners, storage, software licences.
  • File sharing: multiple users access the same documents; version control is easier.
  • Communication: email, messaging, video calls, collaboration platforms.
  • Centralised backups: backups can be scheduled and managed on a server.
  • Remote working: users connect to systems without being physically present.
  • Collaboration: shared documents and shared databases support teamwork.
Exam tip: Whenever you say “resource sharing”, give one example: “shared printer” or “shared folder/server”.

Drawbacks (what can go wrong)

  • Security threats: hacking, phishing, malware, unauthorised access.
  • Data privacy risks: intercepted Wi-Fi traffic, weak passwords.
  • Setup cost: cables, switches, routers, access points, technicians.
  • Maintenance: updates, monitoring, troubleshooting, replacing hardware.
  • Network failure: work may stop if core device fails.
  • Malware spread: shared drives and email can spread infections fast.
High scoring phrase: “A network increases the attack surface because more devices are connected and share data.”

Mini case studies (exam style)

School network

Benefits include shared printers, shared folders for homework, centralised user accounts, and teacher monitoring. Drawbacks include students attempting unauthorised access, viruses spreading via USB or downloads, and disruption when the main switch or router fails.

Hospital network

Benefits include fast access to patient records, sharing imaging results, and coordinated scheduling. Drawbacks include serious privacy risks: if security is weak, sensitive patient data could be leaked. Reliability is critical: outages can affect care.

Exam writing toolkit (copy this structure)

Template for a 6–8 mark answer:
• State the point (benefit or drawback)
• Explain why it happens (because…)
• Give an example context (e.g., school/office/hospital)
• Mention a consequence (therefore/so…)

Example: “Networks allow centralised backups because files can be stored on a server and backed up automatically. In a school, this reduces data loss if a student computer fails, so lessons can continue without missing work.”

Top 10 “must know” one-liners

  • LAN: local, privately owned, high speed for resource sharing.
  • WAN: connects LANs over long distances using telecom infrastructure.
  • WLAN: LAN using Wi-Fi radio signals via an access point.
  • Router: connects networks; forwards packets using IP addresses.
  • Switch: connects devices in a LAN; forwards frames using MAC addresses.
  • Hub: broadcasts to all ports; less efficient and less secure.
  • MAC: unique hardware address of a NIC used in local delivery.
  • IP: logical address used for routing across networks.
  • DNS: converts domain names to IP addresses.
  • Cloud: services (storage/apps) delivered over the Internet.

Most asked comparisons

Switch vs hub (classic)

A switch forwards data only to the intended device using MAC addresses, reducing collisions and improving security. A hub broadcasts data to all devices, causing collisions and allowing other devices to see traffic.

Router vs switch

A router connects different networks and routes packets using IP addresses. A switch connects devices within the same LAN and forwards frames using MAC addresses.

IP vs MAC

An IP address is used for routing across networks and can change (e.g., assigned by DHCP). A MAC address is linked to the NIC hardware and is used for local network delivery.

Star vs bus (faults)

In star, one cable failure affects one device; but central device failure stops the network. In bus, the backbone is a single point of failure and collisions can occur.
