4.1 Types of Networks

LAN

Area
Small (room/building/campus)
Ownership
Usually private (one organisation)
Speed
Typically high
Common media
Ethernet, Wi‑Fi
Exam sentence

A LAN connects devices in a small area to share data and resources at high speed.

WAN

Area
Large (cities/countries/continents)
Ownership
Uses telecom/ISP infrastructure
Speed
Often lower than LAN, higher delay
Example
The Internet (largest WAN)
Exam sentence

A WAN connects networks over long distances, often using leased lines, fibre or satellite links.

WLAN

A WLAN is a LAN that uses Wi‑Fi (radio waves) for local connections.

  • + Mobility (users can move)
  • − Interference (walls, other signals)
  • − Security risks if Wi‑Fi encryption is weak

Typical question (6–8 marks)

Compare LAN and WLAN and justify when WLAN is better.

WLAN is better when mobility matters (e.g., tablets in class) because users can move without cables, but security must be managed (WPA2/WPA3).

4.2 Network Hardware

Router vs Switch vs Hub (quick memory)

  • Router: connects networks, uses IP to forward packets.
  • Switch: connects devices in a LAN, uses MAC to forward frames to the right port.
  • Hub: broadcasts to all ports (inefficient and insecure).

Device “one-liners” (mark-scheme style)

  • NIC: provides network connectivity; has a unique MAC address.
  • Modem: connects to the ISP by converting signals for transmission.
  • Switch: reduces congestion by sending data only to destination device.
  • Router: routes packets between networks using IP addresses.
Common mistake: “Modem and router are the same.” In answers: modem = ISP connection; router = routes between networks.

4.4 Internet Concepts

IP vs MAC (fast comparison)

IP address
Logical address used for routing between networks
MAC address
Hardware address used inside a LAN
Changes?
IP can change; MAC usually stays the same
Memory hook: IP = home address; MAC = device serial number.

DNS in one diagram (idea)

Domain name → DNS → IP address → connect

If DNS fails, names won’t resolve even if the Internet link is working.

Cloud computing (what to evaluate)

  • Access anywhere + collaboration
  • Internet dependency
  • Security/privacy + subscriptions

4.5 Benefits and Drawbacks of Networks

Benefits (write “because…”)

  • Resource sharing because one printer can be used by many users
  • Collaboration because shared documents can be edited by teams
  • Centralised backup because data can be backed up from one server
  • Remote access because users can connect securely from home

Drawbacks (link to mitigation)

  • Security threats → use authentication, encryption, firewalls
  • Cost → plan equipment and scale gradually
  • Maintenance → network admin, updates, monitoring
  • Failure impact → redundancy, backups, UPS
